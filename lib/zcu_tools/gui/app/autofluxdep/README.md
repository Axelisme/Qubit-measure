# gui/app/autofluxdep/ — autofluxdep-gui app shell

控制型 GUI：拿某 flux 的 base context 衍生、自動跑 flux × node-graph 掃描。**domain core**（Builder/Node/orchestrator/Result/Patch/Plotter 的設計語言）見 `CONTEXT.md` + ADR-0018；本 note 記 **app shell**（session 服務組合 + dialog + run path）。

## 結構

```
autofluxdep/
├── controller.py    — Controller façade：組 build_session_services + 實作 SessionControllerPort（委派 conn/ctx/dev/startup/progress/predictor）；workflow 編輯命令；start_run/stop_run/dry_run；RUN 是 OperationRunner client（app-local RUN kind + cooperative cancel + shared progress）；_build_tools（exp_context.predictor→FluxoniumPredictorAdapter / None→SimplePredictor）；_MlModuleSource（ModuleLibrary→ModuleSource proxy，None-on-absent + forward make_cfg/get_waveform）
├── state.py         — AutoFluxDepState(SessionState)：nodes/flux_values/flux_device_name/run_results/run_predictor + ProjectInfo；version keys workflow/flux
├── app.py           — build_core(project?, project_root?) + run_app（注入 repo-root project_root，組 MainWindow）
├── events/          — autofluxdep domain payloads（`run.py`：Run* 生命週期；`workflow.py`：Workflow/Flux 編輯）；`EventBus` 本身只是 `BaseEventBus` 的別名（app.py/controller.py `import BaseEventBus as EventBus`，無獨立 event_bus.py）
├── operation_gate.py— app-local OperationGate（str-keyed 衝突矩陣 over session kinds + 自己 RUN kind）
├── cfg/             — 唯一 import measure spec/value 模型的 seam：`__init__`（純資料：CfgSchema/*Spec/*Value re-export）+ `schema`（NodeCfgSchema：node 旋鈕 SSOT，sectioned value tree + flat logical-key projection，lower/set_field/with_overrides/read_knobs/read_value_tree）+ `form`（CfgFormWidget/SectionLiveField/LiveModelEnv re-export，UI seam）
├── services/        — app-local services：`caretaker`/`persistence_types` 保存 workflow memento；`remote/` 是 **read-only RPC bridge（第二 View，ADR-0013）**：service（RemoteControlAdapter 綁 base scaffolding）/ dispatch（6 pure-query handler）/ method_specs（Qt-free wire 契約，MCP server 直接 import 生 tool schema）/ events（payload-type→requery serializer）/ wire_version（WIRE=1/GUI=1）。agent 只觀測、user 在 GUI 驅動
├── ui/              — main_window（measure-gui style top session row：context/setup/predictor status + Setup/Devices/Predictor/Inspect buttons；持單一 non-modal InspectDialogBase 生命週期）/ node_list（workflow list + flux sweep expression input + value-source helper + flux-source picker + Run/Stop）/ node_detail / node_cfg_form（typed cfg form：SectionLiveField over PlacedNode.schema + CfgFormWidget，右側欄位 label 使用較寬 elide 寬度；編輯經 controller.set_node_params 寫回 schema SSOT）
├── nodes/           — **domain core（別動）**：builder（Builder/Node/RunEnv）/ qubit_freq/lenrabi/ro_optimize/t1/t2ramsey/t2echo/mist/predictor / module_aliases（workflow logical module name → measure-gui calibrated ML fallback names）/ waveform_defaults（named waveform or inline const fallback）/ acquire（共用 real-acquire helpers：set_flux_by_name/require_flux_device/build_stop_checkers/is_good_fit/parse_linear_axis）/ result / plotters / io / spec
├── orchestrator.py / derivation.py / tools.py / registry.py — **domain core（別動）**
└── CONTEXT.md / README.md
```

## 關鍵設計

- **複用 session core**：Controller `__init__` 組 `build_session_services`，注入 app-local `OperationGate` + 共用 off-main 執行器 `BackgroundRunner`（`gui/background.py`，`submit(work,*,run_in_pool,on_done,on_error[,enter])`；owner 直接持具體 runner 才能呼 `quiesce()`，session service 只認 `BackgroundExecutor` port 的 `submit`，ADR-0026 §2）+ 共用 `OperationHandles`/`ProgressService`(default `QtProgressTransport`)/`OperationRunner`(kind-agnostic 生命週期機制，ADR-0026 §1；SoCConnectionService/DeviceService/autofluxdep RUN 都是 client)/`IOManager`。`build_session_services(..., runner=...)` 簽名新增 runner（device_registry 省略 → production singleton adapter）。實作 `SessionControllerPort`（pyright 在 dialog call site 驗 conformance），開共用 `gui/session/ui` 的 setup/device/predictor/inspect dialog。session 回的 bundle 含 `soc_connection`（SoCConnectionService）+ `predictor`（PredictorService，自 ConnectionService 拆出）。run 讀 `exp_context`（soc/soccfg/ml/md/predictor），無自己的 SetupResources/setup()（早退）。
- **inspect（Phase 160c）**：Inspect button → `MainWindow._on_inspect` 開共用 `InspectDialogBase`（**直接用，不 subclass**——measure 才有的 ml create/modify 綁 CfgEditor，刻意排除）。non-modal `open()`（event loop 持續 pump，run worker 不阻塞）+ `WA_DeleteOnClose`，MainWindow 持單一 instance（重開只 raise）。md 讀/改/刪 + ml view/rename/delete 全委派 controller 的 inspect port，落到共用 `ContextService`——它 mutate 後 emit `Md/Ml/ContextSwitched` 到**同一條** `BaseEventBus`，dialog 訂閱故自動刷新（**無需 emit 缺口補丁**：controller 純委派，emit 已在 session core）。Inspect button **run 中不鎖**（inspector 反映 live context，可中途窺看）。
- **State main-thread 不變式 + Qt 需求**：`build_core()` 建 session-service QObject，**需 QApplication 先存在**（測試 conftest `qapp` autouse）；headless 測試經 `connect_mock`（async SoCConnectionService + QEventLoop）建 mock soc，不用真 setup。
- **RUN lifecycle + Run path（real acquire）**：RUN 走 shared `OperationRunner`：controller 在主線程提交 `OperationSpec`，用 app-local `OperationKind.RUN` 佔 hardware exclusion，`OperationHandles.stop(token, reason)` 觸發 cooperative cancel，`ProgressService` 綁一條 flux-sweep progress bar；terminal callback 只做領域判讀（finished/cancelled/failed）與 run event。domain flux loop 仍在 `Orchestrator`：每 node `produce` 用 `Builder.make_cfg`（決策 A，D1：在 produce 跑）lower context→真 cfg，把這點 flux 值寫進 `cfg.dev[flux_device]`（`set_flux_by_name`，by device **name**＝picker 存的，非下層 `flux_dev` label）→ `setup_devices` 推送 → 建 program（TwoToneProgram/ModularProgramV2…）`.acquire`（`round_hook`+`stop_checkers`：cooperative stop＋SNR early-stop，會把當前 row 的 running-average SNR 寫入 Result 供 liveplot title 顯示）→ fit（qubit_freq 並 `predictor.calibrate`；mist/ro_optimize 無 fit）。cancel terminal 是 `RunStoppedPayload`/operation `cancelled`，保留已完成 partial `run_results`；produce 例外仍轉 `RunFailedPayload`。**無 synthetic fallback**：context 未配置時 `make_cfg` **Fast Fail**（→ orchestrator 轉 `RunFailedPayload`，不讓 worker thread abort）。offline 測試走 flux-aware MockSoc（`connect_mock` 起 `fake_flux`），資料隨 flux 變化。詳見 `CONTEXT.md` 的 "Run path"。
- **ro_optimize landscape**：readout freq × gain SNR map 在 argmax 前使用共用 wavelet smoothing；freq/gain 軸由 `make_cfg()` 的 search window 決定，`produce()` 會在 acquire 前同步 result/plot 軸與實際 sweep；plotter 以 scatter marker 標註目前 row 以前最近一次 finite `best_ro_freq` / `best_ro_gain`。此 node 仍是 no-fit argmax，產出 `best_ro_freq` / `best_ro_gain` / `opt_readout`。
- **produce 例外保護**：`orchestrator.run` catch `produce` 例外 → 記 `self.run_error`＋停 sweep；`controller.start_run` 檢查後 emit `RunFailedPayload(message)`，UI `MainWindow._on_run_failed` 解鎖＋彈 warning。真 GUI run 的 node 例外不會 crash app。
- **predictor 兩層**：`exp_context.predictor` 存 raw `FluxoniumPredictor`（PredictorDialog/`PredictorService` 載——predictor seam 自 ConnectionService 拆進 PredictorService，ADR-0026 §5）；`_build_tools` 每 run wrap 成自適應 `FluxoniumPredictorAdapter`（None→`SimplePredictor`），stash 進 run-lived `state.run_predictor`。**勿混** measure 的 freq-預測 predictor。
- **node 參數 SSOT（Phase 160b）**：`PlacedNode` 持自己的 `NodeCfgSchema`（per-placement 可變 value 樹，從 `builder.make_default_schema()` 建，建構時 `overrides=` 種子）——`params: dict`/`base_params` 已刪。real node builders 宣告 sectioned UI value tree（例如 Sweep/Acquisition/Drive pulse），但 `NodeCfgSchema` 對 builder/runtime/remote 保持 flat logical-key projection；`lower()` / `set_field()` / `with_overrides()` / `read_knobs()` 都以 stable logical keys 工作，`read_value_tree()` 才是 UI/debug 用 nested reader。schema defaults 是 operator-ready 初始值（採 notebook / measure-gui 慣例的 waveform 名稱、channel fallback、early-stop 門檻與 sweep window），typed leaves 可用 shared expression mode / `@{...}` value-source helper；lower 時以 active `MetaDict` 立即求值，並支援 sweep edge expression。使用者清空 optional 欄位後 lower 仍會省略，讓 `make_cfg` 在缺少必要實體設定時 Fast Fail；若只有 named waveform asset 不存在，drive/mist pulse 會用對應 length 建 inline const waveform，支援 measure-gui mock 初始化出的 module-only context。orchestrator 把 `provider.schema` 帶進 `RunEnv.schema`，node `make_cfg`/`make_init_result`/`detune_ratio`/`earlystop_snr` 都從 schema lower（`env.schema.lower(ml, md=env.md)`；result allocation 用 active md），推導欄位仍由 produce 注入（不變）。UI typed form 編輯先由 `NodeCfgSchema.logical_updates_from(schema.value)` 把 sectioned draft 投影回 logical-key updates，再經 `controller.set_node_params(index, {key: leaf})`（單一 typed 入口，接受 value-tree leaf 或 raw，fast-fail 未知 key）寫回 schema SSOT，主執行緒寫 + bump workflow version。
- **workflow persistence**：對齊 measure-gui 的 Memento + Caretaker：`Controller` 是 originator，`services/caretaker.py` 只負責單檔 JSON I/O（`autofluxdep_state_v1.json`，user cache 的 gui 目錄），啟動時 restore、`MainWindow.closeEvent` flush。memento 保存 Nodes（type/name + `NodeCfgSchema.schema` 經 shared `session_codec` 的 raw shape）、flux sweep（三個 expression 字串 + 最後 commit 的 explicit values）與 app-shell UI preference（auto-follow active tab）；不保存 run result、plot、predictor、SoC、device live selection。單一壞 node restore 會被 reject 並保留其它 node。
- **ModuleLibrary fallback aliases**：workflow 內部的 module keys 維持 domain logical name（`readout`、`opt_readout`、`pi_pulse`、`pi2_pulse`），但 `ModuleDep.aliases` 讓 library fallback 使用 measure-gui calibrated names：readout 類找 `readout_dpm → readout_rf → readout → res_readout`，pi 類找 `pi_amp/pi_len`，pi2 類找 `pi2_amp/pi2_len/pi_amp/pi_len`。Node-produced module 仍永遠優先於 ML fallback。
- **Produced module contract**：workflow module stream 只流通可被 `ModuleCfgFactory` lower 的 concrete module；lenrabi 產出的 `pi_pulse` / `pi2_pulse` 是從 lowered drive pulse 複製並覆寫 waveform length 的 `type: "pulse"` module，不使用 `type: "pi"` / `"pi2"` 這類 domain placeholder。
- **flux sweep input**：左側 flux start/stop/npts 復用 shared `ScalarLiveField` + `ScalarWidget`，支援 direct value 與 explicit expression mode、ghost preview、`@{...}` value-source completion/space-to-resolve helper；Run 前以 active md resolve 成 `np.linspace`，三個 expression 字串會進 workflow memento。
- **auto-follow tab policy**：run 中進入新的 workflow node 時，左側 selection 會跟著切到該 node 並顯示 run tab；如果進入的是目前已選取的 node，保留使用者當下的 edit/run sub-tab，不打斷正在查看的設定頁。左側 checkbox 可在 run 中即時開關此自動 selection/tab switch；重新打開時會立刻跟到目前 active node，且 preference 會保存。run 中若使用者手動切換 node 或 edit/run tab，auto-follow 會自動取消並同步取消 checkbox，避免下一個 node 事件把視圖拉走。

- **read-only remote bridge（Phase 160e）**：`services/remote/` 是 autofluxdep 的第二 View（RPC face onto Controller，ADR-0013），對標 fluxdep/dispersive bridge。完全 read-only——agent 只觀測、user 在 GUI 驅動 workflow，**無 stop、零 mutation**。三 lifecycle（launch/connect/disconnect）+ 5 純查詢：`state.check`（readiness flags + predictor 旗標）/`project.info`（chip/qub/result_dir/params_path，None→四欄 null）/`workflow.list`（每 placed node 的 name/type/provides/requires/has_result，**不含 predictor service**——run 時才 prepend）/`node.cfg(name)`（單 node 未 lower 的 user knobs，仍是 flat logical-key projection；sweep→`{start,stop,expts}`，走 `NodeCfgSchema.read_knobs()`）/`result.summary`（每有 Result 的 node 的 n_flux/n_measured/tiny fit_summary，**非 raw 2D**）。`resources.versions` 只是 MCP/RPC optimistic-concurrency bookkeeping，存在於 registry/METHOD_SPECS，不是 agent-facing pure query tool。app 接線走 app-local 就地啟停 adapter（`run_app(control=)`，鏡像 `run_qt_app` 的三行；不動共用層、不需 `ensure_host`——worker 用 Agg 不畫圖）。所有 handler 走 base 主執行緒 marshal；run 中讀 `run_results` 拿一致快照（worker 原地填 numpy 非 State semantic write）。MCP server 在 `lib/zcu_tools/mcp/autofluxdep/`（port 8768），skill `run-autofluxdep-gui`。

跨模組設計見 ADR-0013（RemoteControlAdapter=第二 View）/0017（worker 畫圖 marshal，本 app 不適用——worker 不畫）/0018（orchestrator 需求解析器）/0020（session-core extraction）/0026（operation abstraction：OperationRunner + scope-as-adapter-input + ConnectionService 拆 SoC/Predictor + DeviceRegistryPort）。
