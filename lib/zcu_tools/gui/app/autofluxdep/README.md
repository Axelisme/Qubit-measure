# gui/app/autofluxdep/ — autofluxdep-gui app shell

控制型 GUI：拿某 flux 的 base context 衍生、自動跑 flux × node-graph 掃描。**domain core**（Builder/Node/orchestrator/Result/Patch/Plotter 的設計語言）見 `CONTEXT.md` + ADR-0018；本 README 記 **app shell**（session 服務組合 + dialog + run path）。

## 結構

```
autofluxdep/
├── controller.py    — Controller façade：組 build_session_services 並暴露 `setup_control` / `context_control` / `device_control` / `predictor_control` / `progress_control` facets；project apply handoff 同步 ProjectInfo 並從 `fluxdep_fit` best-effort auto-load predictor；workflow 編輯命令；start_run/stop_run/dry_run；RUN 是 OperationRunner client（app-local RUN kind + cooperative cancel + shared progress）；begin_shutdown 使用 shared QtShutdownDriver cancel/poll live operations，quiesce_background 在 window teardown 前 drain BackgroundRunner；_build_tools（exp_context.predictor→FluxoniumPredictorAdapter / None→SimplePredictor）；_MlModuleSource（ModuleLibrary→ModuleSource proxy，None-on-absent + forward make_cfg/get_waveform）

├── app.py           — build_core(project?, project_root?) + run_app（注入 repo-root project_root，組 MainWindow）
├── events/          — autofluxdep domain payloads（`run.py`：Run* 生命週期；`workflow.py`：Workflow/Flux 編輯）；`EventBus` 本身只是 `BaseEventBus` 的別名（app.py/controller.py `import BaseEventBus as EventBus`，無獨立 event_bus.py）

├── cfg/             — 唯一 import measure spec/value 模型的 seam：`__init__`（純資料：CfgSchema/*Spec/*Value re-export）+ `schema`（NodeCfgSchema：node 旋鈕 SSOT，raw make_cfg-shaped value tree + flat logical-key projection，lower/set_field/with_overrides/read_knobs/read_value_tree）+ `form`（CfgFormWidget/SectionLiveField/LiveModelEnv re-export，UI seam）

├── nodes/           — **domain core（別動）**：builder（Builder/Node/RunEnv）/ qubit_freq/lenrabi/ro_optimize/t1/t2ramsey/t2echo/mist/predictor / module_aliases（workflow logical module name → measure-gui calibrated ML fallback names）/ waveform_defaults（named waveform or inline const fallback）/ acquire（共用 real-acquire helpers：set_flux_by_name/require_flux_device/build_stop_checkers/is_good_fit/axis_to_sweep）/ result / plotters / io / spec
├── orchestrator.py / derivation.py / tools.py / registry.py — **domain core（別動）**
└── CONTEXT.md / README.md
```

## 關鍵設計

- **複用 session core**：Controller `__init__` 組 `build_session_services`，注入 app-local `OperationGate` + 共用 off-main 執行器 `BackgroundRunner`（`gui/background.py`，`submit(work,*,run_in_pool,on_done,on_error)`；owner 直接持具體 runner 才能呼 `quiesce()`，session service 只認 `BackgroundExecutor` port 的 `submit`，ADR-0026 §2）+ 共用 `OperationHandles`/`ProgressService`(default `QtProgressTransport`)/`OperationRunner`(kind-agnostic 生命週期機制，ADR-0026 §1；SoCConnectionService/DeviceService/autofluxdep RUN 都是 client)/`IOManager`。`build_session_services(..., runner=..., on_project_applied=...)` 接收 runner 與 project hook（device_registry 省略 → production singleton adapter）；hook 讓共用 SetupDialog apply 後回到 autofluxdep 同步 shell `ProjectInfo` 並嘗試讀 `fluxdep_fit` 安裝 predictor。並把 session bundle 的 `SetupControlPort` / `ContextControlPort` / `DeviceControlPort` / `PredictorControlPort` 暴露給 shared setup/inspect/device/predictor dialog 與 md providers，把 `ProgressControlPort` 暴露給 app-local run progress widgets；開共用 `gui/session/ui` 的 setup/device/predictor/inspect dialog。session 回的 bundle 含 `soc_connection`（SoCConnectionService）+ `predictor`（PredictorService）+ `setup_control`（SetupDialog composition facade）+ `context_control`（context switching/md/ml/value-source/bind-device new-context facet）+ `device_control`（device lifecycle/query/progress/event-subscription facet）+ `predictor_control`（predictor lifecycle/query/compute/event-subscription facet）+ `progress_control`（owner-keyed run progress facet）。run 讀 `exp_context`（soc/soccfg/ml/md/predictor），無自己的 SetupResources/setup()。
- **inspect dialog**：Inspect button → `MainWindow._on_inspect` 開共用 `InspectDialogBase(self._ctrl.context_control, self._ctrl.get_bus())`（**直接用，不 subclass**——measure 才有的 ml create/modify 綁 CfgEditor，刻意排除）。non-modal `open()`（event loop 持續 pump，run worker 不阻塞）+ `WA_DeleteOnClose`，MainWindow 持單一 instance（重開只 raise）。md 讀/改/刪 + ml view/rename/delete 全委派 `ContextControlPort`，落到共用 `ContextService`——它 mutate 後 emit `Md/Ml/ContextSwitched` 到**同一條** `BaseEventBus`，dialog 訂閱故自動刷新。Inspect button **run 中不鎖**（inspector 反映 live context，可中途窺看）。
- **State main-thread 不變式 + Qt 需求**：`build_core()` 建 session-service QObject，**需 QApplication 先存在**（測試 conftest `qapp` autouse）；headless 測試經 `connect_mock`（async SoCConnectionService + QEventLoop）建 mock soc，不用真 setup。
- **RUN lifecycle + Run path（real acquire）**：RUN 走 shared `OperationRunner`：controller 在主線程提交 `OperationSpec`，用 app-local `OperationKind.RUN` 佔 hardware exclusion，`OperationHandles.stop(token, reason)` 觸發 cooperative cancel，`ProgressService` 綁一條 flux-sweep progress bar；UI 另以 `ProgressStack` 顯示目前 node/flux row 的 `rounds` 進度與 ETA（過濾掉已由底部 bar 顯示的 flux sweep）。terminal callback 只做領域判讀（finished/cancelled/failed）與 run event，shared runner 會隔離 terminal/progress cleanup 例外以保證 handle settle 與 lease release。worker-originated point/node/predictor-refresh run events 先經 controller-owned Qt relay 回到主線程，再呼 `BaseEventBus.emit()`；`_RunBridge` 只把 main-thread domain payload 投影成 UI signal。domain flux loop 仍在 `Orchestrator`：每 node `produce` 用 `Builder.make_cfg` lower context→真 cfg，把這點 flux 值寫進 `cfg.dev[flux_device]`（`set_flux_by_name`，by device **name**＝picker 存的，非下層 `flux_dev` label）→ `setup_devices` 推送 → 建 program（TwoToneProgram/ModularProgramV2…）`.acquire`（`round_hook`+`stop_checkers`：cooperative stop＋SNR early-stop，會把當前 row 的 running-average SNR 寫入 Result 供 liveplot title 顯示）→ fit（qubit_freq 並 `predictor.calibrate`；mist/ro_optimize 無 fit）。worker round-hook 送到 `_RunBridge` 的 row redraw 是 coalesced live-frame：同一 node/flux row 若已有 queued update，只保留 dirty flag，main thread 畫完後最多補畫一次最新 state，避免重畫過期 frame；這條高頻 redraw path 不進 EventBus；plotter redraw 例外會 log 並停用該 node plotter，不讓 Qt slot abort app。同步 start 失敗會清掉 run-lived `run_results` / `run_predictor`，避免 remote `has_results` 回報未開始的 stale products；cancel terminal 是 `RunStoppedPayload`/operation `cancelled`，保留已完成 partial `run_results`；produce 例外仍轉 `RunFailedPayload`。**無 synthetic fallback**：context 未配置時 `make_cfg` **Fast Fail**（→ orchestrator 轉 `RunFailedPayload`，不讓 worker thread abort）。offline 測試走 flux-aware MockSoc（`connect_mock` 起 `fake_flux`），資料隨 flux 變化。詳見 `CONTEXT.md` 的 "Run path"。
- **Run Result Artifact（ADR-0040）**：每次 production run 需要 project，並建立兩個同 slug root：`ProjectInfo.result_dir/autofluxdep_runs/<timestamp_slug>/` 是輕量 metadata root（`manifest.json` / `journal.jsonl` / `report.md`），`ProjectInfo.database_path/autofluxdep_runs/<timestamp_slug>/` 是 heavy data root（`nodes/*.hdf5` / `exports/fluxdep/*.hdf5`；`database_path` 使用 shared startup resolved value，通常是 dated `Database/<chip>/<qub>/YYYY/MM/Data_MMDD`）。`manifest.json` 保存 `format_version=1`、run identity、workflow hash、project `result_dir`/`database_path`、metadata/data roots、node file list、exports/reports 與 terminal status；node/export paths 相對 data root，journal/report paths 相對 metadata root。run artifact 的 workflow snapshot 只包含本次 run 的 enabled runnable node subset；disabled node 留在 GUI workflow memento 但不配置 Result/HDF5、不參與 dependency/provides。`journal.jsonl` 是 `event_version=1` append-only row audit，記錄 `node_row_written`、`node_skipped`、`node_failed`、`flux_committed`、`run_failed`、`run_finalized`。每個有 Result 的 enabled placed node 有一個 streaming grouped Labber HDF5，node row 在 `produce` 正常返回、Patch validate、且 journal payload strict JSON preflight 成功後才寫入並 flush；stop/failed run 仍 finalize artifact，保留已 committed rows。`qubit_freq` terminal export 只使用 journal-committed node rows，並把 row-local absolute frequency (`predict_freq + detune`) resample 到 common absolute frequency grid，再輸出 fluxdep raw Labber spectrum；markdown report 只在 terminal finalize 產生，PNG/MP4/resume/browser 不在 A1。
- **qubit_freq drive feedback**：`qubit_freq` 成功 fit 會回報 raw `fit_kappa`，並另回報 `qfw_factor = fit_kappa / drive_gain`。下一個 flux point 的 `make_cfg` 預設讀 orchestrator 以 `step_weighted` 投影的 smoothed `qfw_factor`，用 notebook 同款 `min(max_drive_gain, target_kappa / qfw_factor)` 產生 drive gain；第一點沒有 feedback 時可從 md `qf_w` 與 seed gain 推估 `qfw_factor`，或沿用 schema 中的 fixed `qub_gain`。node cfg 的 generation override 可把 drive gain mode 切到 fixed，讓所有點直接使用 `qub_gain`。
- **RUN profiling probes**：`ZCU_AUTOFLUXDEP_PROFILE=1`（或將對應 logger 設為 DEBUG）啟用低侵入 timing logs。probe 只觀測不改排程：Qt main thread 量 `row_updated` queue delay、`plotter.update()`、progress render（含 `ProgressStack` derive / Qt property writes 拆段）、plot/canvas build；worker side 量每個 node `produce()`、共用 1-D `round_hook` processing、`ro_optimize` program build / acquire / SNR landscape，以及 MockSoc SimEngine 的 `poll_data` / `compute_round` / signal-grid / population-chain（含 cooperative GIL yield gap）。log prefix 為 `autofluxdep profile`，含 thread 標籤，用來先定位卡頓在 main-thread redraw、progress render、或 worker-side hidden compute / GIL starvation，再決定優化策略。
- **generation overrides**：autofluxdep node 的上方 Default cfg 保持實驗 cfg shape，下方 Generation overrides 只描述跨 flux 的生成策略；其內部用 nested behavior groups（`sweep` / `timing` / `feedback` / `fit` / `safety`）整理欄位，但 builder/runtime/remote 與 persisted workflow/run workflow snapshot 仍讀寫 flat logical keys。T1/T2/echo 預設用 `t1_seed_us` / `t2*_seed_us` 加 factor 產生 relax delay 與 sweep stop；lenrabi 是以 `expected_pi_length` 為 setpoint 的 drive-gain 負回饋控制：下一點 gain 使用 smoothed/seed `pi_product` 除以 `pi_product_factor * expected_pi_length`，上一點 measured `pi_length` 只用於 auto sweep range；每個可信點仍產出 raw `pi_product = measured_pi_length * applied_gain` 供後續 flux point 平滑。ro_optimize 用 T1 seed 設 relax delay，readout freq/gain window 可在 fixed sweep 與 previous-best centered window 間切換；t2echo 可依 detune 自動選 fringe/decay fit 或手動固定 fit method。
- **ro_optimize landscape**：readout freq × gain SNR map 在 argmax 前使用共用 wavelet smoothing；freq/gain 軸由 `make_cfg()` 的 search window 決定，`produce()` 會在 acquire 前同步 result/plot 軸與實際 sweep；fresh default 以 md `r_f` 或 readout module freq/gain 為中心，使用窄窗做 first-pass search。plotter 以 scatter marker 標註目前 row 以前最近一次 finite `best_ro_freq` / `best_ro_gain`。此 node 仍是 no-fit argmax，產出 `best_ro_freq` / `best_ro_gain` / `opt_readout`。
- **produce 例外保護**：`orchestrator.run` catch `produce` 例外 → 記 `self.run_error`＋停 sweep；`controller.start_run` 檢查後 emit `RunFailedPayload(message)`，UI `MainWindow._on_run_failed` 解鎖＋彈 warning。真 GUI run 的 node 例外不會 crash app。
- **predictor 兩層 + live view**：`exp_context.predictor` 存 raw `FluxoniumPredictor`（PredictorDialog/`PredictorService` 載——predictor seam 自 ConnectionService 拆進 PredictorService，ADR-0026 §5）；setup project 若 resolved `params.json` 含 `fluxdep_fit`，controller best-effort 安裝 predictor，使用者仍可在 PredictorDialog typed path/Browse 載入或手動覆蓋；`_build_tools` 每 run wrap 成自適應 `FluxoniumPredictorAdapter`（None→`SimplePredictor`），stash 進 run-lived `state.run_predictor`。run 中 Predictor toolbar button 保持可開，既有 shared `PredictorDialog` 進 `set_live_mode(True)`：模型欄位、Load/Browse/Apply、device-value picker 與 marker drag 變 read-only，marker 由目前 flux point 的 device value 驅動；qubit_freq row committed 後 controller 只 emit shared `PredictorChangedPayload` 讓 dialog 重讀 raw predictor bias / redraw curves，dialog 不讀 run-local IDW correction。Run finish/stop/fail 會退出 live mode。**勿混** measure 的 freq-預測 predictor。
- **node 參數 SSOT**：`PlacedNode` 持自己的 `NodeCfgSchema`（per-placement 可變 value 樹，固定從 `builder.make_default_schema(ctx)` 建，fresh placement 只用 active `ExpContext.md/ml` 種一次預設值，建構時 `overrides=` 再覆寫；persisted workflow restore 仍以保存的 raw value tree 為準，不用新的 context default 覆寫舊設定）；workflow 不使用獨立 `params: dict` / `base_params`。real node builders 的 Default cfg 使用對應 measure-gui adapter 的 cfg spec 與 copied value tree，再套 node-local autofluxdep overlay / default overrides，因此 UI 顯示 adapter 原生 `modules.*` / `sweep.*` / scalar fields（含 `ModuleRef` / `WaveformRef` shape），但 fresh defaults 可對齊 md notebook 的 sweep/rounds/relax/gain 慣例。autofluxdep 只在外層追加 `generation` section，並在 section 內用 behavior groups 分類；`logical_paths` 把 adapter-shaped value tree 投影成 builder/runtime/remote/persistence 使用的 stable logical keys；`lower()` 逐 logical path 解析 scalar/sweep/ref leaf，`lower_raw(ml, md)` 則 lower 完整 adapter-shaped cfg 並移除整個 `generation`。node `make_cfg` 以 `lower_raw()` 為基底，僅注入 flux workflow 需要的 derived 值（例如 predictor frequency、latest produced readout/pi pulse、auto/fixed range/relax/gain 行為），不再重建 adapter 預設 cfg/spec。UI typed form 編輯把 Default cfg 與 Generation overrides 的完整 draft value tree 經 `controller.set_node_cfg_value()` 寫回 schema SSOT，主執行緒寫 + bump workflow version；materialized cfg editor 會像 measure-gui 一樣在 md/ml/context/device session event 後刷新 EvalValue/ref snapshot；舊的 `set_node_params()` 仍只作為 logical-key 測試/程式入口。
- **workflow persistence**：對齊 measure-gui 的 Memento + Caretaker：`Controller` 是 originator，`services/caretaker.py` 只負責單檔 JSON I/O（`autofluxdep_state_v1.json`，user cache 的 gui 目錄），workflow/node/flux/UI preference edits 會 debounce autosave，`MainWindow.closeEvent` 會先經 shared shutdown driver cancel/poll live operations，再 flush persistence、quiesce background runner 與 teardown；RUN finished/stopped/failed terminal callback 會立即 flush，避免長掃描或 stop 後遺失 workflow edits；capture/write 失敗會轉成 typed persistence failure 並由 controller 記錄，不讓 Qt timer slot abort GUI。memento 保存 shared startup prefill（scope id / chip-qubit-res / connection / remembered devices）、Nodes（type/name/enabled + `NodeCfgSchema.to_persisted_raw()`；Default cfg 走 shared `session_codec` raw shape，`generation` 以 flat logical-key raw map 保存）、flux sweep（三個 expression 字串 + 最後 commit 的 explicit values）與 app-shell UI preference（auto-follow active tab）；不保存 run result、plot、predictor、SoC、device live selection。restore 只 seed setup dialog prefill 與 remembered devices，不自動套用 project 或連線；單一壞 node restore 會被 reject 並保留其它 node。
- **ModuleLibrary fallback aliases**：workflow 內部的 module keys 維持 domain logical name（`readout`、`opt_readout`、`pi_pulse`、`pi2_pulse`），但 `ModuleDep.aliases` 讓 library fallback 使用 measure-gui calibrated names：readout 類找 `readout_dpm → readout_rf → readout → res_readout`，pi 類找 `pi_amp/pi_len`，pi2 類找 `pi2_amp/pi2_len/pi_amp/pi_len`。Node-produced module 仍永遠優先於 ML fallback。
- **Produced module contract**：workflow module stream 只流通可被 `ModuleCfgFactory` lower 的 concrete module；lenrabi 產出的 `pi_pulse` / `pi2_pulse` 是從 lowered drive pulse 複製並覆寫 waveform length 的 `type: "pulse"` module，不使用 `type: "pi"` / `"pi2"` 這類 domain placeholder。lenrabi fit 先比較 decay/non-decay Rabi fit 並只在 pi length、boundary、residual gate 可信時送出 feedback Patch；不可信點保留 raw Result row 但不產生 `pi_length` / `pi_product` / drive modules。pi2 過短或 non-finite 時仍送出 `pi_length` / `pi_product` scalar feedback，但不產生 `pi2_length` 或任何 drive module，避免下游混用不同 flux point 的 pulse pair。
- **flux sweep input**：左側 flux start/stop/npts 復用 shared `ScalarLiveField` + `ScalarWidget`，支援 direct value 與 explicit expression mode、ghost preview、`@{...}` value-source completion/space-to-resolve helper；Run 前以 active md resolve 成 `np.linspace`，三個 expression 字串會進 workflow memento。Run button 只有在 setup 完成、workflow 至少一個 enabled node、且 flux source 已選定時啟用；缺任一條件時在點擊前 block。左側 node checkbox 只控制 future run membership，disabled node 仍可選取與編輯 cfg。
- **auto-follow tab policy**：run 中進入新的 workflow node 時，左側 selection 會跟著切到該 node 並顯示 run tab；如果進入的是目前已選取的 node，保留使用者當下的 edit/run sub-tab，不打斷正在查看的設定頁。左側 checkbox 可在 run 中即時開關此自動 selection/tab switch；重新打開時會立刻跟到目前 active node，且 preference 會保存。auto-follow navigation 只更新 selection/title/run canvas；edit form 在使用者打開 edit tab 或 run done 回 edit 時才 lazy materialize，避免 run 中每次 active-node transition 都重建 hidden typed form。run 期間已 materialize 的 node edit form 會以 node/index 快取並重用；run 結束時清掉 hidden form cache，回到一般 workflow 編輯語意。run 中若使用者手動切換 node 或 edit/run tab，auto-follow 會自動取消並同步取消 checkbox，避免下一個 node 事件把視圖拉走。
- **run progress render**：run 進度由共用 `ProgressStack` 顯示 per-node rounds，底部 global bar 顯示 flux sweep；兩者都保留 profiling probe。高頻 progress event 只在 maximum/format/value 實際變更時更新 lightweight progress widget，`setValue` 只寫狀態並排程 repaint，避免主線程卡在 native `QProgressBar.setValue`。

- **read-only remote bridge**：`services/remote/` 是 autofluxdep 的第二 View（RPC face onto Controller，ADR-0013），對標 fluxdep/dispersive bridge。完全 read-only——agent 只觀測、user 在 GUI 驅動 workflow，**無 stop、零 mutation**。三 lifecycle（launch/connect/disconnect）+ 5 純查詢：`state.check`（readiness flags + predictor 旗標）/`project.info`（chip/qub/result_dir/database_path/params_path，None→五欄 null）/`workflow.list`（每 placed node 的 name/type/enabled/provides/requires/has_result，**不含 predictor service**——run 時才 prepend；disabled node 仍列出但 future run 不執行）/`node.cfg(name)`（單 node 未 lower 的 user knobs，仍是 flat logical-key projection；sweep→`{start,stop,expts}`，走 `NodeCfgSchema.read_knobs()`）/`result.summary`（每有 Result 的 node 的 n_flux/n_measured/tiny fit_summary，**非 raw 2D**）。`resources.versions` 只是 MCP/RPC optimistic-concurrency bookkeeping，存在於 registry/METHOD_SPECS，不是 agent-facing pure query tool。app 接線走 app-local 就地啟停 adapter（`run_app(control=)`，鏡像 `run_qt_app` 的三行；不動共用層、不需 `ensure_host`——worker 用 Agg 不畫圖）。所有 handler 走 base 主執行緒 marshal；run 中讀 `run_results` 拿一致快照（worker 原地填 numpy 非 State semantic write）。MCP server 在 `lib/zcu_tools/mcp/autofluxdep/`（port 8768），skill `run-autofluxdep-gui`。

跨模組設計見 ADR-0013（RemoteControlAdapter=第二 View）/0017（worker 畫圖 marshal，本 app 不適用——worker 不畫）/0018（orchestrator 需求解析器）/0020（session-core extraction）/0026（operation abstraction：OperationRunner + scope-as-adapter-input + ConnectionService 拆 SoC/Predictor + DeviceRegistryPort）。
