# gui/app/autofluxdep/ — autofluxdep-gui app shell

控制型 GUI：拿某 flux 的 base context 衍生、自動跑 flux × node-graph 掃描。**domain core**（Builder/Node/orchestrator/Result/Patch/Plotter 的設計語言）見 `CONTEXT.md` + ADR-0018；本 README 記 **app shell**（session 服務組合 + dialog + run path）。

## 結構

```
autofluxdep/
├── controller.py    — Controller façade：組 build_session_services 並暴露 `setup_control` / `context_control` / `device_control` / `predictor_control` / `progress_control` facets；project apply handoff 同步 ProjectInfo 並從 `fluxdep_fit` best-effort auto-load predictor；workflow 編輯命令；start_run/request_pause/continue_run/stop_run/dry_run；RUN 是 OperationRunner client（app-local RUN kind + shared progress）；begin_shutdown 使用 shared QtShutdownDriver cancel/poll live operations，quiesce_background 在 window teardown 前 drain BackgroundRunner；_build_tools（exp_context.predictor→FluxoniumPredictorAdapter / None→SimplePredictor）；_MlModuleSource（ModuleLibrary→ModuleSource proxy，None-on-absent + forward make_cfg/get_waveform）
```

## 關鍵設計

- **複用 session core**：Controller `__init__` 組 `build_session_services`，注入 app-local `OperationGate` + 共用 off-main 執行器 `BackgroundRunner`（`gui/background.py`，`submit(work,*,run_in_pool,on_done,on_error)`；owner 直接持具體 runner 才能呼 `quiesce()`，session service 只認 `BackgroundExecutor` port 的 `submit`，ADR-0026 §2）+ 共用 `OperationHandles`/`ProgressService`(default `QtProgressTransport`)/`OperationRunner`(kind-agnostic 生命週期機制，ADR-0026 §1；SoCConnectionService/DeviceService/autofluxdep RUN 都是 client)/`IOManager`。`build_session_services(..., runner=..., on_project_applied=...)` 接收 runner 與 project hook（device_registry 省略 → production singleton adapter）；hook 讓共用 SetupDialog apply 後回到 autofluxdep 同步 shell `ProjectInfo` 並嘗試讀 `fluxdep_fit` 安裝 predictor。Controller 以 `run_locks.py` guarded wrappers 暴露 `SetupControlPort` / `ContextControlPort` / `DeviceControlPort` / `PredictorControlPort` 給 shared setup/inspect/device/predictor dialog 與 md providers：run/paused 時 read/query 仍可用，但 setup/context/device/predictor mutation Fast Fail；MainWindow 進入 run/pausing/paused 時會關閉已開的 Setup/Devices dialog，避免 timer 或按鈕持續碰 setup/device lifecycle。`ProgressControlPort` 仍直接暴露給 app-local run progress widgets。session 回的 bundle 含 `soc_connection`（SoCConnectionService）+ `predictor`（PredictorService）+ `setup_control`（SetupDialog composition facade）+ `context_control`（context switching/md/ml/value-source/bind-device new-context facet）+ `device_control`（device lifecycle/query/progress/event-subscription facet）+ `predictor_control`（predictor lifecycle/query/compute/event-subscription facet）+ `progress_control`（owner-keyed run progress facet）。run 讀 `exp_context`（soc/soccfg/ml/md/predictor），無自己的 SetupResources/setup()。
- **inspect dialog**：Inspect button → `MainWindow._on_inspect` 開共用 `InspectDialogBase` 的 app-local read-only-aware instance（measure 才有的 ml create/modify 綁 CfgEditor，刻意排除）。non-modal `open()`（event loop 持續 pump，run worker 不阻塞）+ `WA_DeleteOnClose`，MainWindow 持單一 instance（重開只 raise）。idle 時 md set/delete + ml rename/delete 委派 guarded `ContextControlPort`，落到共用 `ContextService` 並 emit `Md/Ml/ContextSwitched`；running/pausing/paused 時 Inspect button 仍可開來看 live context，但 edit fields 與 Set/Delete/Rename/Delete 按鈕禁用，port wrapper 也拒絕直接 mutation。
- **State main-thread 不變式 + Qt 需求**：`build_core()` 建 session-service QObject，**需 QApplication 先存在**（測試 conftest `qapp` autouse）；headless 測試經 `connect_mock`（async SoCConnectionService + QEventLoop）建 mock soc，不用真 setup。
- **RUN lifecycle + Run path（real acquire）**：RUN 走 shared `OperationRunner`，但 run-lived state 由 `RunSession` 持有。controller 在主線程建立 session 並提交每個 run/continue segment 的 `OperationSpec`，用 app-local `OperationKind.RUN` 佔 hardware exclusion，`ProgressService` 綁一條 flux-sweep progress bar；UI 另以 `ProgressStack` 顯示目前 node/flux row 的 `rounds` 進度與 ETA。Pause 是 flux-boundary non-terminal state：`RunSession.request_pause()` 讓 worker 完成當前 flux point、commit rows/flux、flush artifact，operation settle 並釋放 hardware lease，session 保留 `InfoStore` / `Tools` / `SmoothingService` / feedback / predictor / writers / results；Continue 開新 operation 從原始 `next_flux_idx` 繼續，所以 EWMA / step-weighted smoothing 不會在 pause boundary 重新 seed。Continue 是 same-process only；沒有跨 process resume/browser。Stop/Abort 是 terminal cancel/finalize，不能 continue；若 stop 在 operation token 建立後、worker closure 啟動前發生，`RunSession.prepare_segment()` 之後不再由 worker 清掉 stop flag，所以 cancellation 會在 flux 0 前被 honor。Restart 會先把 paused artifact finalize 為 stopped、丟掉 paused session，再從 flux 0 建立 fresh run。worker-originated point/node/predictor-refresh run events 先經 controller-owned Qt relay 回到主線程，再呼 `BaseEventBus.emit()`；`_RunBridge` 只把 main-thread domain payload 投影成 UI signal。domain flux loop 仍在 `Orchestrator`：每 node `produce` 用 `Builder.make_cfg` lower context→真 cfg，把這點 flux 值寫進 `cfg.dev[flux_device]`（`set_flux_by_name`，by device **name**＝picker 存的，非下層 `flux_dev` label）→ `setup_devices` 推送 → 建 program（TwoToneProgram/ModularProgramV2…）`.acquire`（`round_hook`+`stop_checkers`：cooperative stop＋SNR early-stop，會把當前 row 的 running-average SNR 寫入 Result 供 liveplot title 顯示）→ fit。**無 synthetic fallback**：context 未配置時 `make_cfg` **Fast Fail**（→ orchestrator 轉 `RunFailedPayload`，不讓 worker thread abort）。offline 測試走 flux-aware MockSoc（`connect_mock` 起 `fake_flux`），資料隨 flux 變化。詳見 `CONTEXT.md` 的 "Run path"。
- **Run Result Artifact（ADR-0040/0043）**：每次 production run 需要 project，並建立兩個同 slug root：`ProjectInfo.result_dir/autofluxdep_runs/<timestamp_slug>/` 是輕量 metadata root（`manifest.json` / `journal.jsonl` / `report.md`），`ProjectInfo.database_path/autofluxdep_runs/<timestamp_slug>/` 是 heavy data root（`nodes/*.hdf5` / `exports/fluxdep/*.hdf5`；`database_path` 使用 shared startup resolved value，通常是 dated `Database/<chip>/<qub>/YYYY/MM/Data_MMDD`）。`manifest.json` 保存 `format_version=1`、run identity、workflow hash、project `result_dir`/`database_path`、metadata/data roots、node file list、exports/reports、terminal status，以及 `lifecycle.status` / `next_flux_idx`；node/export paths 相對 data root，journal/report paths 相對 metadata root。run artifact 的 workflow snapshot 只包含本次 run 的 enabled runnable node subset；每個 workflow node entry 同時保存 editable persisted `cfg`/`cfg_hash` 與 run-start truth `base_cfg`/`override_plan`。disabled node 留在 GUI workflow memento 但不配置 Result/HDF5、不參與 dependency/provides。`journal.jsonl` 是 `event_version=1` append-only row audit，記錄 `node_row_written`、`node_skipped`、`node_failed`、`flux_committed`、`run_paused`、`run_continued`、`run_failed`、`run_finalized`。每個有 Result 的 enabled placed node 有一個 streaming grouped Labber HDF5，node row 在 `produce` 正常返回、Patch validate、且 journal payload strict JSON preflight 成功後才寫入並 flush；pause 不 finalize、不 close writers；stop/failed run 仍 finalize artifact，保留已 committed rows。`qubit_freq` terminal export 只使用 journal-committed node rows，並把 row-local absolute frequency (`predict_freq + detune`) resample 到 common absolute frequency grid，再輸出 fluxdep raw Labber spectrum；markdown report 只在 terminal finalize 產生，PNG/MP4/cross-session resume/browser 不在本 app shell contract。
- **qubit_freq feedback**：`qubit_freq` 用分層 gate 保護兩條 feedback path。頻率 fit 通過基本 residual gate 時會填 `qubit_freq` / `fit_detune`，並把 measured-minus-base residual observe 到 generic feedback estimator。`bias_update_mode` 預設是 `fixed`：physical/base predictor 不變，下一點 drive center 由 node 自己組合 `base predict_freq + confidence * estimated correction`，result 的 `predict_freq` role 記錄實際 center，長時間沒有可信 fit 時 correction 依 `pred_freq_correction_decay_points` 平滑退回 0；fresh node 的 pred_freq IDW default 為 `k=10`、`epsilon=1e-4`、`decay_points=4`。`bias_update_mode="hard"` 才會在可信 fit 後呼 `predictor.calibrate`，並以校準後 base prediction 計 residual。linewidth feedback 需再通過較嚴格 residual gate、finite/positive FWHM 與 detune window bound，才回報 raw `fit_kappa` 與 `qfw_factor = fit_kappa / drive_gain`。下一個 flux point 的 drive gain 仍預設讀 orchestrator 以 `step_weighted` 投影的 smoothed `qfw_factor`，用 notebook 同款 `min(max_drive_gain, target_kappa / qfw_factor)` 產生；中等品質 fit 可幫頻率 prediction correction 但不污染 drive-gain feedback。第一點沒有 qfw feedback 時可從 md `qf_w` 與 seed gain 推估 `qfw_factor`，或沿用 schema 中的 fixed `qub_gain`。node cfg 的 generation override 可把 drive gain mode 切到 fixed，讓所有點直接使用 `qub_gain`。
- **RUN profiling probes**：`ZCU_AUTOFLUXDEP_PROFILE=1`（或將對應 logger 設為 DEBUG）啟用低侵入 timing logs。probe 只觀測不改排程：Qt main thread 量 `row_updated` queue delay、`plotter.update()`、progress render（含 `ProgressStack` derive / Qt property writes 拆段）、plot/canvas build；worker side 量每個 node `produce()`、共用 1-D `round_hook` processing、`ro_optimize` program build / acquire / SNR landscape，以及 MockSoc SimEngine 的 `poll_data` / `compute_round` / signal-grid / population-chain（含 cooperative GIL yield gap）。log prefix 為 `autofluxdep profile`，含 thread 標籤，用來先定位卡頓在 main-thread redraw、progress render、或 worker-side hidden compute / GIL starvation，再決定優化策略。
- **generation overrides + OverridePlan（ADR-0043）**：autofluxdep node 的上方 Default cfg 保持實驗 cfg shape，下方 Generation overrides 只描述跨 flux 的生成策略；其內部用 nested behavior groups（`sweep` / `timing` / `feedback` / `fit` / `safety`）整理欄位，但 builder/runtime/remote 與 persisted workflow/run workflow snapshot 仍讀寫 flat logical keys。Builder 用 `override_plan(schema)` 宣告 Generation 會覆寫的 Default cfg path；`OverridePlan` 只描述 Default cfg path，不描述 `generation.*` 欄位，run start materialize `RunCfgSnapshot(base_cfg, override_plan, knobs)` 並驗證 path 存在於 `base_cfg` 且不是 whole-module replacement。每個 flux point 從 run-start `base_cfg` deep-copy 後只套用 builder-declared patches；undeclared path、first point 的 `after_first_point` patch、缺失 path 都 Fast Fail。Generated module override path 只覆蓋 dependency source 實際接管的 core leaves；所有 autofluxdep readout refs restrict to `Pulse Readout`，使 readout freq/gain/window path 在 UI、OverridePlan 與 runtime patch 中保持同一 shape。`CfgFormWidget` seam 依 full path 套用 `FieldDecoration`；node cfg form 會把 `all_points` 欄位標成 generated、disabled，並把 `after_first_point` 欄位標成 initial，讓 user 知道第一點讀 UI 值、後續點由 Generation 覆寫。T1/T2/echo 預設用 `t1_seed_us` / `t2*_seed_us` 加 factor 產生 relax delay 與 sweep stop；T1/T2/T2Echo scalar feedback 只在 residual gate 與 sweep-window bound 可信時產出，否則保留 raw Result row 但不送出 `t1` / `t2r` / `t2r_detune` / `t2e` Patch key。smoothed dependency projection 會跨 raw-absent 點 carry forward 上一個可信 smoothed 值，但 smoother state 本身不因缺 raw 而 advance。lenrabi 是以 `expected_pi_length` 為 setpoint 的 drive-gain 負回饋控制：第一點或 disabled slot 回到 seed/smoothed `pi_product` 行為；可信 fit 後 generic `log_step` controller 依 `log(measured_pi_length / expected_pi_length)` 對下一點 gain 提 proposal，預設 step gain 0.5，下一點以 confidence 在 log-domain 平滑混合 proposal 與 open-loop `pi_product` gain，長時間沒有可信 fit 時依 `pi_gain_feedback_decay_points` 退回 open-loop；`pi_product_factor` 預設 1.2，max gain clamp 仍由 lenrabi use site 處理；每個可信點仍產出 raw `pi_product = measured_pi_length * applied_gain` 供既有 smoothing/diagnostic 使用。ro_optimize 用 T1 seed 設 relax delay，readout freq/gain window 可在 fixed sweep 與 previous-best centered window 間切換；t2echo 可依 detune 自動選 fringe/decay fit 或手動固定 fit method。
- **ro_optimize landscape**：readout freq × gain SNR map 在 argmax 前使用共用 wavelet smoothing；freq/gain 軸由 `make_cfg()` 的 search window 決定，`produce()` 會在 acquire 前同步 result/plot 軸與實際 sweep；fresh default 以 md `r_f` 或 readout module freq/gain 為中心，使用窄窗做 first-pass search。Patch emission 前會拒絕 invalid/flat/non-unique/boundary/low-prominence landscape；被拒絕的 row 保留 SNR map evidence，但不送出 `best_ro_freq` / `best_ro_gain` / `opt_readout`，讓 downstream 使用最近可信值或既有 fallback。plotter 以 scatter marker 標註目前 row 以前最近一次 finite `best_ro_freq` / `best_ro_gain`。此 node 仍是 no-fit argmax，可信時產出 `best_ro_freq` / `best_ro_gain` / `opt_readout`。
- **produce 例外保護**：`orchestrator.run` catch `produce` 例外 → 記 `self.run_error`＋停 sweep；controller 檢查後 emit `RunFailedPayload(message, node, flux_idx, stage)`，finalize/pause/stop artifact failure 也用同一 payload 帶 `stage`。UI `MainWindow._on_run_failed` 解鎖＋彈 warning；running-close terminal stop path 不重複彈 warning。真 GUI run 的 node 例外不會 crash app。
- **predictor 兩層 + live view**：`exp_context.predictor` 存 raw `FluxoniumPredictor`（PredictorDialog/`PredictorService` 載——predictor seam 自 ConnectionService 拆進 PredictorService，ADR-0026 §5）；setup project 若 resolved `params.json` 含 `fluxdep_fit`，controller best-effort 安裝 predictor，使用者仍可在 PredictorDialog typed path/Browse 載入或手動覆蓋；`_build_tools` 每 run wrap 成 base predictor `FluxoniumPredictorAdapter`（None→`SimplePredictor`），stash 進 run-lived `state.run_predictor`，generic residual correction 另在 `Tools.feedback` 的 qubit_freq slot。run 中 Predictor toolbar button 保持可開，既有 shared `PredictorDialog` 進 `set_live_mode(True)`：模型欄位、Load/Browse/Apply、device-value picker 與 marker drag 變 read-only，marker 由目前 flux point 的 device value 驅動；qubit_freq row committed 後 controller 只 emit shared `PredictorChangedPayload` 讓 dialog 重讀 raw predictor bias / redraw curves，dialog 不讀 run-local correction estimator。fixed-bias default 下 raw predictor dialog 不顯示 run-local residual correction；只有 hard bias mode 的 raw predictor calibration 會反映在 dialog 曲線上。Run finish/stop/fail 會退出 live mode。**勿混** measure 的 freq-預測 predictor。
- **node 參數 SSOT**：`PlacedNode` 持自己的 `NodeCfgSchema`（per-placement 可變 value 樹，固定從 `builder.make_default_schema(ctx)` 建，fresh placement 只用 active `ExpContext.md/ml` 種一次預設值，建構時 `overrides=` 再覆寫；persisted workflow restore 仍以保存的 raw value tree 為準，不用新的 context default 覆寫舊設定）；workflow 不使用獨立 `params: dict` / `base_params`。real node builders 的 Default cfg 使用對應 measure-gui adapter 的 cfg spec 與 copied value tree，再套 node-local autofluxdep overlay / default overrides，因此 UI 顯示 adapter 原生 `modules.*` / `sweep.*` / scalar fields（含 `ModuleRef` / `WaveformRef` shape），但 fresh defaults 可對齊 md notebook 的 sweep/rounds/relax/gain 慣例。autofluxdep 只在外層追加 `generation` section，並在 section 內用 behavior groups 分類；`logical_paths` 把 adapter-shaped value tree 投影成 builder/runtime/remote/persistence 使用的 stable logical keys；`lower()` 逐 logical path 解析 scalar/sweep/ref leaf，`lower_raw(ml, md)` 則 lower 完整 adapter-shaped cfg 並移除整個 `generation`。run start 固定 materialize `base_cfg` / `override_plan` / `knobs`；node `make_cfg` 讀 `RunEnv.knobs()` 並透過 `Builder.point_cfg()` 產生 point cfg，避免每點重新 lower 或整包替換 module。UI typed form 編輯把 Default cfg 與 Generation overrides 的完整 draft value tree 經 `controller.set_node_cfg_value()` 寫回 schema SSOT，主執行緒寫 + bump workflow version；materialized cfg editor 會像 measure-gui 一樣在 md/ml/context/device session event 後刷新 EvalValue/ref snapshot；舊的 `set_node_params()` 仍只作為 logical-key 測試/程式入口。
- **workflow persistence + close behavior**：對齊 measure-gui 的 Memento + Caretaker：`Controller` 是 originator，`services/caretaker.py` 只負責單檔 JSON I/O（`autofluxdep_state_v1.json`，user cache 的 gui 目錄），workflow/node/flux/UI preference edits 會 debounce autosave。`MainWindow.closeEvent` 在一般 live operations 上仍走 shared shutdown driver；running RUN 則先確認、送 terminal stop、等待最多 30 秒，逾時只顯示 explicit Force Close；paused close 需確認 discard/terminal finalize，並把 artifact finalize 為 stopped 後丟棄 in-memory paused session。Force Close 可能只留下最後已 flushed 的 artifact state。RUN finished/stopped/failed terminal callback 會立即 flush，避免長掃描或 stop 後遺失 workflow edits；capture/write 失敗會轉成 typed persistence failure 並由 controller 記錄，不讓 Qt timer slot abort GUI。memento 保存 shared startup prefill（scope id / chip-qubit-res / connection / remembered devices）、Nodes（type/name/enabled + `NodeCfgSchema.to_persisted_raw()`；Default cfg 走 shared `session_codec` raw shape，`generation` 以 flat logical-key raw map 保存）、flux sweep（三個 expression 字串 + 最後 commit 的 explicit values）與 app-shell UI preference（auto-follow active tab）；不保存 run result、plot、predictor、SoC、device live selection。restore 只 seed setup dialog prefill 與 remembered devices，不自動套用 project 或連線；單一壞 node restore 會被 reject 並保留其它 node。
- **ModuleLibrary fallback aliases**：workflow 內部的 module keys 維持 domain logical name（`readout`、`opt_readout`、`pi_pulse`、`pi2_pulse`），但 `ModuleDep.aliases` 讓 library fallback 使用 measure-gui calibrated names：readout 類找 `readout_dpm → readout_rf → readout → res_readout`，pi 類找 `pi_amp/pi_len`，pi2 類找 `pi2_amp/pi2_len/pi_amp/pi_len`。Node-produced module 仍永遠優先於 ML fallback。
- **Produced module contract**：workflow module stream 只流通可被 `ModuleCfgFactory` lower 的 concrete module；lenrabi 產出的 `pi_pulse` / `pi2_pulse` 是從 lowered drive pulse 複製並覆寫 waveform length 的 `type: "pulse"` module，不使用 `type: "pi"` / `"pi2"` 這類 domain placeholder。需要 concrete pulse 的 downstream node 以 required `ModuleDep` 表達需求；沒有 node-produced module 且 ML alias 也找不到時，resolver 直接 journal `node_skipped`，不注入 placeholder。lenrabi fit 先比較 decay/non-decay Rabi fit 並只在 pi length、boundary、residual gate 可信時送出 feedback Patch；不可信點保留 raw Result row 但不產生 `pi_length` / `pi_product` / drive modules。pi2 過短或 non-finite 時仍送出 `pi_length` / `pi_product` scalar feedback，但不產生 `pi2_length` 或任何 drive module，避免下游混用不同 flux point 的 pulse pair。
- **flux sweep input**：左側 flux start/stop/npts 復用 shared `ScalarLiveField` + `ScalarWidget`，支援 direct value 與 explicit expression mode、ghost preview、`@{...}` value-source completion/space-to-resolve helper；Run click 由 controller 以 active md resolve 並 commit 成 explicit `np.linspace` values，三個 expression 字串會進 workflow memento。Run button 以 controller `run_readiness()` 判斷 setup/project/workflow/enabled-node readiness；flux source 可不選，`flux_device_name is None` 表示 bare-number sweep。左側 node checkbox 只控制 future run membership，disabled node 仍可選取與編輯 cfg。node list row 的 checkbox + label 由自訂 row widget 單一繪製，`QListWidgetItem` 只保存 tooltip / data，不放 display text，避免 Qt delegate 與 row label 疊字。
- **auto-follow tab policy**：run 中進入新的 workflow node 時，左側 selection 會跟著切到該 node 並顯示 run tab；如果進入的是目前已選取的 node，保留使用者當下的 edit/run sub-tab，不打斷正在查看的設定頁。左側 checkbox 可在 run 中即時開關此自動 selection/tab switch；重新打開時會立刻跟到目前 active node，且 preference 會保存。auto-follow navigation 只更新 selection/title/run canvas；edit form 在使用者打開 edit tab 或 run done 回 edit 時才 lazy materialize，避免 run 中每次 active-node transition 都重建 hidden typed form。run 期間已 materialize 的 node edit form 會以 node/index 快取並重用；run 結束時清掉 hidden form cache，回到一般 workflow 編輯語意。run 中若使用者手動切換 node 或 edit/run tab，auto-follow 會自動取消並同步取消 checkbox，避免下一個 node 事件把視圖拉走。
- **run progress render**：run 進度由共用 `ProgressStack` 顯示 per-node rounds，底部 global bar 顯示 flux sweep；兩者都保留 profiling probe。高頻 progress event 只在 maximum/format/value 實際變更時更新 lightweight progress widget，`setValue` 只寫狀態並排程 repaint，避免主線程卡在 native `QProgressBar.setValue`。

- **read-only remote bridge**：`services/remote/` 是 autofluxdep 的第二 View（RPC face onto Controller，ADR-0013），對標 fluxdep/dispersive bridge。完全 read-only——agent 只觀測、user 在 GUI 驅動 workflow，**無 stop、continue、restart、close、零 mutation**。三 lifecycle（launch/connect/disconnect）+ 6 agent-facing 純查詢：`state.check`（readiness flags + `is_running` / `is_paused` / `next_flux_idx` / `run_status` + predictor 旗標）/`project.info`（chip/qub/result_dir/database_path/params_path，None→五欄 null）/`workflow.list`（每 placed node 的 name/type/enabled/provides/requires/has_result，**不含 predictor service**——run 時才 prepend；disabled node 仍列出但 future run 不執行）/`node.cfg(name)`（單 node 未 lower 的 user knobs，仍是 flat logical-key projection；sweep→`{start,stop,expts}`，走 `NodeCfgSchema.read_knobs()`；另回傳 builder `override_plan`）/`result.summary`（每有 Result 的 node 的 n_flux/n_measured/tiny fit_summary，**非 raw 2D**；`n_measured` 數 primary raw signal finite rows，`fit_summary.n_fitted` 數 fit scalar finite rows）/`ui.screenshot(target="window")`（只抓主視窗 PNG 到 `/tmp` 並回傳 path/bytes，純觀測）。`resources.versions` 只是 MCP/RPC optimistic-concurrency bookkeeping，存在於 registry/METHOD_SPECS，不是 agent-facing pure query tool。app 接線走 app-local 就地啟停 adapter（`run_app(control=)`，鏡像 `run_qt_app` 的三行；不動共用層、不需 `ensure_host`——worker 用 Agg 不畫圖）。所有 handler 走 base 主執行緒 marshal；run 中讀 `run_results` 拿一致快照（worker 原地填 numpy 非 State semantic write）。MCP server 在 `lib/zcu_tools/mcp/autofluxdep/`（port 8768），skill `run-autofluxdep-gui`。
