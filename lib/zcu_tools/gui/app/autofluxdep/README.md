# gui/app/autofluxdep/ — autofluxdep-gui app shell

控制型 GUI：拿某 flux 的 base context 衍生、自動跑 flux × node-graph 掃描。**domain core**（Builder/Node/orchestrator/Result/Patch/Plotter 的設計語言）見 `CONTEXT.md` + ADR-0018；本 README 記 **app shell**（session 服務組合 + dialog + run path）。

## 結構

```
autofluxdep/
├── controller.py    — Controller façade：組 build_session_services 並暴露 `setup_control` / `context_control` / `device_control` / `predictor_control` / `progress_control` facets；project apply handoff 同步 ProjectInfo 並從 `fluxdep_fit` best-effort auto-load predictor；workflow 編輯命令以單一 helper clear run products + bump version + emit edited node hint + persist；start_run/request_pause/continue_run/stop_run/dry_run；RUN 是 OperationRunner client（app-local RUN kind + shared progress）；begin_shutdown 使用 shared QtShutdownDriver cancel/poll live operations，quiesce_background 在 window teardown 前 drain BackgroundRunner；run provider/tools/result/cfg snapshot/session 建構委派給 `services/run_setup.py`
├── run_session.py   — one in-memory run session：持有 run-lived `RunStore` / `Tools` / `SmoothingService` / providers / `InfoStore` / cursor / open writers / Result objects；每個 run/continue segment 是一個 OperationRunner operation；Pause 是 flux-boundary non-terminal state，Stop/Abort 是 terminal finalize
├── run_locks.py     — guarded wrappers over shared setup/context/device/predictor control ports；run/paused 時允許 read/query 與 device live refresh，但拒絕 setup/context/device/predictor mutation；device live refresh 以 `try_poll_device_info() -> bool` 顯式回報 inner refresh result
├── state.py         — AutoFluxDepState(SessionState)：nodes/flux_values/flux_device_name/run_results/run_predictor + ProjectInfo；version keys workflow/flux
├── app.py           — build_core(project?, project_root?) + AutoFluxDepGuiBehavior（runtime behavior；launcher 注入 repo-root project_root，組 MainWindow）
├── events/          — autofluxdep domain payloads（`run.py`：Run* 生命週期；`workflow.py`：Workflow/Flux 編輯）；`EventBus` 本身只是 `BaseEventBus` 的別名（app.py/controller.py `import BaseEventBus as EventBus`，無獨立 event_bus.py）
├── operation_gate.py— app-local OperationGate thin wrapper（注入 autofluxdep RUN kind 到 shared RunBlocksHardwareGate）
├── cfg/             — generic Spec/Value/spec walk/inheritance/codec/lowering與`CfgSchemaAssembler`由caller直接從`zcu_tools.gui.cfg`匯入，program shape/spec與missing/nested/reference/subset materialization policy取自Qt-free `gui.measure_cfg`；local package barrel只暴露autoflux-owned `NodeCfgSchema`、OverridePlan/policy、module reference spec helpers與其它local API；`NodeSchemaBuilder`只把paired tree mechanics交給shared assembler，logical paths、compound declarations、module seeding、generation verbs與section labels仍留在autoflux domain；module/waveform共用`ReferenceSpec(kind=...)` / `ReferenceValue`，`lowering` 依opaque kind組autoflux-local expression/reference/range ports，`module_adapter`只做runtime normalization與autoflux policy binding，不import measure app；autoflux policy讓Arb data維持free-form string且不啟用readout inheritance，只materialize Pulse/Pulse Readout module，但reference catalog先辨識完整合法shape，corrupt entry Fast Fail；`NodeCfgSchema` 擁有 node 旋鈕 SSOT、raw make_cfg-shaped value tree、flat logical-key projection、lower/set_field/with_overrides/read_knobs/to_persisted_raw/restore_persisted_raw（no-knob builder 用 `empty_node_schema()`）；`override_plan` 擁有 Default cfg runtime override contract（RunCfgSnapshot / path validation / declared patch application；見 ADR-0043/0046/0051）；本package不擁有Qt form seam
├── feedback/        — generic scalar feedback/control capability（ADR-0041/0042）：run start 依 placed-node slot declarations 建 estimator/controller state，掛在 `Tools.feedback` 並以 `RunEnv.feedback` 給 node；估測/控制回傳 `FeedbackSample(value, confidence, age_queries)`，confidence 依 query age 平滑衰減；不處理 gates/clamp/safety/fallback
├── services/        — app-local services：`caretaker`/`persistence_types` 保存 workflow memento；`artifact_paths` 集中 artifact slug/relative path helpers；`run_store`/`result_io`/`fluxdep_export`/`labber_browser_export`/`sample_table_export`/`run_report` 擁有 Run Result Artifact（ADR-0040/0043：result_dir metadata root、database_path data root、manifest、journal、per-node streaming Labber HDF5、live fixed-axis Labber Browser sidecars、terminal fluxdep/qubit_freq export 與 report，以及使用者/script 觸發的 artifact-derived sample table export；non-terminal pause 寫 `lifecycle` + journal 但不 close writer；workflow node snapshot 保存 `cfg`/`cfg_hash` + run-start `base_cfg`/`override_plan`）；`remote/` 是 **read-only RPC bridge（第二 View，ADR-0013）**：service（RemoteControlAdapter 綁 base scaffolding + optional screenshot view）/ dispatch（7 pure-query handler）/ method_specs（Qt-free wire 契約，MCP server 直接 import 生 tool schema）/ events（payload-type→requery serializer）/ wire_version（WIRE=5/GUI=8）。agent 只觀測、user 在 GUI 驅動
├── ui/              — main_window（measure-gui style top session row：context/setup/predictor status + Setup/Devices/Predictor/Inspect buttons；running close 需確認並送 terminal stop，等 30s 後才顯示 explicit Force Close；paused close 需確認並 finalize stopped；Setup/Devices/Inspect non-modal dialog lifetime 由 DialogRefStore 保存，run/paused 時 Devices 保持可開但切 read-only inspection，Predictor dialog 另有 typed live-mode handle；md/ml/context/device session event 會刷新 materialized node cfg editor；run 中只立即 redraw 目前顯示的 node canvas，hidden node row update 記 latest dirty idx，切到該 node 才 lazy refresh；terminal finished/stopped 後保持目前 edit/run sub-tab，並可透過 off-main Export sample 另存 notebook-style `SampleTable` CSV）/ node_list（workflow list node enable checkbox + flux sweep expression input + value-source helper + flux-source picker + Run status label + Run/Pause/Continue 主按鈕 + paused Restart 同列按鈕 + Stop terminal 按鈕 + terminal Export sample 按鈕）/ node_detail / node_cfg_form（直接import `zcu_tools.gui.widgets.cfg` typed form：把 PlacedNode.schema 拆成 Default cfg 與 Generation overrides 兩個等高 CfgFormWidget；Default cfg 依 builder OverridePlan 顯示 generated/initial badge、tooltip 與 disabled state；Generation overrides 內部用 domain behavior groups，欄位 visible label 可省略 group 已提供的 prefix，strategy/mode group 以 shared `ChoiceSectionSpec` 只渲染當前 choice 需要的欄位；右側欄位 label 使用較寬 elide 寬度；編輯經 controller.set_node_cfg_value 寫回完整 value tree）
├── experiments/     — 使用者可修改的量測實驗：七個 experiment 各自以同名 `.py` 作唯一 authoritative definition，檔尾 export `EXPERIMENT`；`catalog.py` 是唯一顯式 ordered composition root，順序只控制新增選單；`_support/` 擁有至少兩個 experiments 共用的 acquire、Result、Plotter、defaults、schema 與 cfg mechanics。新增或修改實驗先讀 `experiments/README.md`
├── nodes/           — ADR-0018 穩定 execution contracts：`builder.py`（Builder/Node/RunEnv/PlacedNode）、`io.py`（Snapshot/Patch）、`spec.py`（Dependency/ModuleDep）、同一 execution seam 的 pure-compute `predictor.py`，以及窄 package 說明；不放量測實驗或其共用 mechanics
├── orchestrator.py / derivation.py / tools.py — **domain core（別動）**
└── CONTEXT.md / README.md
```

`NodeCfgSchema`的typed Spec/Value path resolve/read/replace直接使用`zcu_tools.gui.cfg.tree`；
generation persistence的raw dict reshape/traversal仍由autofluxdep擁有，不下沉到shared core。

ModuleLibrary reference enumeration與lowering以canonical root-only inspection取得shape，不呼
`to_dict()`或materializer。只有resolve legal且policy discriminator set允許的Pulse/Pulse Readout或
waveform時才呼app materializer façade；其它legal module只回canonical label與`None` value。

## 關鍵設計

- **qubit_freq cfg ownership**：app-local template 直接組合 `ProgramV2Cfg`、
  完整 local module cfg 與 `ExpCfgModel`，讓 node 的 runtime contract 在使用處完整可見。
- **Experiment authoring boundary**：量測 cfg/schema、Node、fit、Patch 與 experiment-specific policy 位於 `experiments/<name>.py`；跨至少兩個實驗的機械 helper 才進 `experiments/_support/`。`experiments/catalog.py` 是 concrete experiment 的唯一 composition root；catalog order 不重排 persisted workflow。Builder/Node contract 與 predictor 留在 `nodes/`，使用者新增實驗時不修改 orchestrator。
- **RUN acquire helper contract**：`experiments/_support/acquire.py` 的 SNR early-stop 是
  completed-round closure；threshold 讀 run-start knob snapshot，probe 尚未有 signal
  時回 False，有 signal 後才交給 SNR checker 判斷，不保存額外 data-stop flag。
  experiment node 自己 inline picked flux device 寫入、Schedule/ProgramBuilder acquire、
  mode 分支、fit dispatch 與 Patch emission；共用 helper 只保留跨 node 且語意精確的機械原語。
- **RunStore terminal finalize**：terminal export、markdown report 與
  `completed_flux_count` 共用同一份 journal snapshot；helper 不自行 fallback
  reread journal，避免 finalize/report 看到不同 committed-row 視圖。
- **複用 session core**：Controller `__init__` 組 `build_session_services`，注入 app-local `OperationGate`（thin wrapper over shared `RunBlocksHardwareGate`）+ 共用 off-main 執行器 `BackgroundRunner`（`gui/background.py`，`submit(work,*,run_in_pool,on_done,on_error)`；owner 直接持具體 runner 才能呼 `quiesce()`，session service 只認 `BackgroundExecutor` port 的 `submit`，ADR-0026 §2）+ 共用 `OperationHandles`/`ProgressService`(default `QtProgressTransport`)/`OperationRunner`(kind-agnostic 生命週期機制，ADR-0026 §1；SoCConnectionService/DeviceService/autofluxdep RUN 都是 client)/`IOManager`。`build_session_services(..., runner=..., on_project_applied=...)` 接收 runner 與 project hook（device_registry 省略 → production singleton adapter）；hook 讓共用 SetupDialog apply 後回到 autofluxdep 同步 shell `ProjectInfo` 並嘗試讀 `fluxdep_fit` 安裝 predictor。Controller 以 `run_locks.py` guarded wrappers 暴露 `SetupControlPort` / `ContextControlPort` / `DeviceControlPort` / `PredictorControlPort` 給 shared setup/inspect/device/predictor dialog 與 md providers：run/paused 時 read/query 仍可用，但 setup/context/device/predictor mutation Fast Fail；MainWindow 進入 run/pausing/paused 時會關閉已開的 Setup dialog，Devices dialog 保持開啟並切 read-only inspection，Refresh/poll 讀取路徑與 `try_poll_device_info()` 保留，device mutation 仍由 `run_locks.py` guard Fast Fail。`ProgressControlPort` 仍直接暴露給 app-local run progress widgets。session 回的 bundle 含 `soc_connection`（SoCConnectionService）+ `predictor`（PredictorService）+ `setup_control`（SetupDialog composition facade）+ `context_control`（context switching/md/ml/value-source/bind-device new-context facet）+ `device_control`（device lifecycle/query/progress/event-subscription facet）+ `predictor_control`（predictor lifecycle/query/compute/event-subscription facet）+ `progress_control`（owner-keyed run progress facet）。run 讀 `exp_context`（soc/soccfg/ml/md/predictor），無自己的 SetupResources/setup()。
- **inspect dialog**：Inspect button → `MainWindow._on_inspect` 開共用 `InspectDialogBase` 的 app-local read-only-aware instance（measure 才有的 ml create/modify 綁 CfgEditor，刻意排除）。non-modal `open()`（event loop 持續 pump，run worker 不阻塞）+ `WA_DeleteOnClose`，MainWindow 持單一 instance（重開只 raise）。idle 時 md set/delete + ml rename/delete 委派 guarded `ContextControlPort`，落到共用 `ContextService` 並 emit `Md/Ml/ContextSwitched`；running/pausing/paused 時 Inspect button 仍可開來看 live context，但 edit fields 與 Set/Delete/Rename/Delete 按鈕禁用，port wrapper 也拒絕直接 mutation。
- **State main-thread 不變式 + Qt 需求**：`build_core()` 建 session-service QObject，**需 QApplication 先存在**（測試 conftest `qapp` autouse）；headless 測試經 `connect_mock`（async SoCConnectionService + QEventLoop）建 mock soc，不用真 setup。
- **RUN lifecycle + Run path（real acquire）**：RUN 走 shared `OperationRunner`，但 run-lived state 由 `RunSession` 持有。controller 在主線程建立 session 並提交每個 run/continue segment 的 `OperationSpec`，用 app-local `OperationKind.RUN` 佔 hardware exclusion，`ProgressService` 綁一條 flux-sweep progress bar；UI 另以 `ProgressStack` 顯示目前 node/flux row 的 `rounds` 進度與 ETA。Pause 是 flux-boundary non-terminal state：`RunSession.request_pause()` 讓 worker 完成當前 flux point、commit rows/flux、flush artifact，operation settle 並釋放 hardware lease，session 保留 `InfoStore` / `Tools` / `SmoothingService` / feedback / predictor / writers / results；Continue 開新 operation 從原始 `next_flux_idx` 繼續，所以 EWMA / step-weighted smoothing 不會在 pause boundary 重新 seed。Continue 是 same-process only；沒有跨 process resume/browser。Stop/Abort 是 terminal cancel/finalize，不能 continue；若 stop 在 operation token 建立後、worker closure 啟動前發生，`RunSession.prepare_segment()` 之後不再由 worker 清掉 stop flag，所以 cancellation 會在 flux 0 前被 honor。Restart 會先把 paused artifact finalize 為 stopped、丟掉 paused session，再從 flux 0 建立 fresh run。worker-originated point/node/predictor-refresh run events 先經 controller-owned Qt relay 回到主線程，再呼 `BaseEventBus.emit()`；`_RunBridge` 只把 main-thread domain payload 投影成 UI signal。domain flux loop 仍在 `Orchestrator`：每 node `produce` 用 `Builder.make_cfg` lower context→真 cfg，把這點 flux 值寫進 `cfg.dev[flux_device]`（`set_flux_by_name`，by device **name**＝picker 存的，非下層 `flux_dev` label）→ `setup_devices` 推送 → 透過 `experiments/_support/acquire.py` helper 建立 `Schedule` / `ProgramBuilder` 執行 node-level program build/acquire（`SignalBuffer` update＋completed-round SNR `stop_condition`；cooperative cancel 走 ambient Schedule stop flag，SNR early-stop 會把當前 row 的 running-average SNR 寫入 Result 供 liveplot title 顯示）→ fit。`generation.acquisition.acquire_retry` 是 real-acquire node 的 program build/acquire retry 預算，預設 3；cfg lowering 與 device setup Fast Fail 且不 retry。`RunSession` 以獨立 user-stop event 與 Schedule-stop event 區分使用者取消與 acquire 失敗，避免 internal Schedule failure 被誤報為 stopped。**無 synthetic fallback**：context 未配置時 `make_cfg` **Fast Fail**（→ orchestrator 轉 `RunFailedPayload`，不讓 worker thread abort）。offline 測試走 flux-aware MockSoc（`connect_mock` 起 `fake_flux`），資料隨 flux 變化。詳見 `CONTEXT.md` 的 "Run path"。
- **Run Result Artifact（ADR-0040/0043）**：每次 production run 需要 project，並建立兩個同 slug root：`ProjectInfo.result_dir/autofluxdep_runs/<timestamp_slug>/` 是輕量 metadata root（`manifest.json` / `journal.jsonl` / `report.md`），`Database/<chip>/<qub>/autofluxdep_runs/<timestamp_slug>/` 是 heavy data root（`nodes/*.hdf5` / `exports/fluxdep/*.hdf5` / `exports/sample/samples.csv` / `labber/YYYY/MM/Data_MMDD/*.hdf5`；若 shared startup 給的是 dated raw `database_path`，RunStore 會剝掉尾端 `YYYY/MM/Data_MMDD` 再建立 autofluxdep run root）。`manifest.json` 保存 `format_version=1`、run identity、workflow hash、project `result_dir`/`database_path`、metadata/data roots、node file list、exports/reports、terminal status，以及 `lifecycle.status` / `next_flux_idx`；node/export paths 相對 data root，journal/report paths 相對 metadata root。run artifact 的 workflow snapshot 只包含本次 run 的 enabled runnable node subset；每個 workflow node entry 同時保存 editable persisted `cfg`/`cfg_hash` 與 run-start truth `base_cfg`/`override_plan`。disabled node 留在 GUI workflow memento 但不配置 Result/HDF5、不參與 dependency/provides。`journal.jsonl` 是 `event_version=1` append-only row audit，記錄 `node_row_written`、`node_skipped`、`node_failed`、`flux_committed`、`run_paused`、`run_continued`、`run_failed`、`run_finalized`。每個有 Result 的 enabled placed node 有一個 streaming grouped Labber HDF5，node row 在 `produce` 正常返回、Patch validate、且 journal payload strict JSON preflight 成功後才寫入並 flush；pause 不 finalize、不 close writers；stop/failed run 仍 finalize artifact，保留已 committed rows。Run start 也預建固定軸 Labber Browser sidecars：`lenrabi`/`mist` 輸出 signal sidecar，`t1`/`t2ramsey`/`t2echo` 輸出 signal 與 fit scalar sidecar，node row commit 後同步寫入並 flush；`ro_optimize` 不輸出 Labber Browser sidecar。`qubit_freq` 維持 terminal export：terminal 時由 journal-committed rows 產生 fluxdep raw Labber spectrum，並在 Labber Browser dated folder 產生等價 single-log sidecar，因為 absolute Frequency 軸需由 committed `predict_freq` 推導。Export sample 是使用者觸發的 notebook-style `SampleTable` CSV，從 `flux_committed` 成功點匯出 `calibrated mA`、`Freq (MHz)`、T1/T2r/T2e、error 欄位與 `date`，不匯出 Rabi length 或 comment；輸出預設 append 到既有 CSV，script 可用 `--overwrite` 重建檔案；standalone `script/export_autofluxdep_sample_table.py` 可對既有 run directory 或 `manifest.json` 執行同一匯出。`exports["fluxdep_spectrum"]` 保持既有語意，`exports["labber_browser_root"]` 與 `exports["labber_browser_sidecars"]` 指向 Labber Browser sidecars；markdown report 只在 terminal finalize 產生，PNG/MP4/cross-session resume/result browser UI 不在本 app shell contract。
- **qubit_freq feedback**：`qubit_freq` 用分層 gate 保護頻率 correction、physical recovery 與 linewidth feedback。Default cfg 的 detune sweep 以 centered value model 呈現：`center=0 MHz` 是 generated/locked，使用者編輯 `span` / `expts`；fresh default 對應到 lower 後的 `-50..50 MHz`、`201` 點，維持 0.5 MHz resolution。頻率 fit 通過基本 residual gate 時會填 `qubit_freq` / `fit_detune`，並把 measured-minus-current-physical residual observe 到 generic feedback estimator；poor fit 仍回傳 empty `Patch()`，但在 `physical_recovery_mode="fail_triggered_fit"` 時會用 run-lived、placement-scoped trusted history 嘗試 physical recovery。raw/base predictor 在 `qubit_freq` 中不被 mutate，且不再提供直接校準 raw predictor 的模式；`make_cfg()` 先取 predictor base 或 accepted physical overlay，再疊加 optional `confidence * pred_freq_correction`。Generation UI 只暴露 `physical_recovery_mode`；fit point 數、candidate center shift 與 RMS guardrails 使用 `DEFAULT_PHYSICAL_RECOVERY_*` constants，並不作為 persisted logical keys。enabled 時 first fail 會用既有 trusted history fit，first success after fail 會先 append 該 trusted point 再 fit，repeated fail 不重複 fit。fit history 先以 latest trusted point wins 正規化 unique flux，少於 10 個 unique flux 不 fit，超過 30 點時以適當最小 flux spacing 選擇不超過 30 且盡量接近 30 個分布點；accepted fit 需 finite、RMS 不比 active physical base 差、RMS 不超過 fixed max RMS，current center shift 不超過 fixed max center shift。accepted overlay 只存在於本 run / placement，不 mutate raw predictor；若 generic correction estimator 可用，會以 accepted overlay 對 selected history 重算 residual 並 `replace_observations(...)`，若 estimator 被關閉則 overlay 仍可接受、只跳過 residual reseed；predictor overlay 建立與 candidate prediction 失敗視為 adapter/model contract error，直接 Fast Fail。fresh node 的 pred_freq IDW default 為 `k=10`、`epsilon=1e-4`、`decay_points=4`，長時間沒有可信 fit 時 correction 依 `pred_freq_correction_decay_points` 平滑退回 0。linewidth feedback 需再通過較嚴格 residual gate、finite/positive FWHM 與 detune window bound，才回報 raw `fit_kappa` 與 `qfw_factor = fit_kappa / drive_gain`。下一個 flux point 的 drive gain 預設讀 orchestrator 以 `step_weighted` 投影的 smoothed `qfw_factor`，用 `min(1.0, target_kappa / qfw_factor)` 產生；中等品質 fit 可幫頻率 prediction correction 但不污染 drive-gain feedback。第一點直接使用 Default cfg 的 Probe Pulse gain 作為 initial value；第二點起才由 adaptive linewidth feedback 產生 drive gain overlay。node cfg 的 generation override 可把 drive gain mode 切到 fixed，讓所有點直接使用 `qub_gain`。
- **RUN profiling probes**：`ZCU_AUTOFLUXDEP_PROFILE=1`（或將對應 logger 設為 DEBUG）啟用低侵入 timing logs。probe 只觀測不改排程：Qt main thread 量 `row_updated` queue delay、`plotter.update()`、progress render（含 `ProgressStack` derive / Qt property writes 拆段）、plot/canvas build；worker side 量每個 node `produce()`、共用 1-D `round_hook` processing、Schedule-backed program build/acquire、`ro_optimize` SNR landscape。log prefix 為 `autofluxdep profile`，含 thread 標籤，用來先定位卡頓在 main-thread redraw、progress render、或 worker-side node/acquire work，再決定優化策略。
- **generation overrides + OverridePlan（ADR-0043）**：autofluxdep node 的上方 Default cfg 保持實驗 cfg shape，下方 Generation overrides 只描述跨 flux 的生成策略；其內部用 domain behavior groups（如 acquisition、relax、sweep、drive_gain、freq_recovery、predictor_correction、pi_feedback、freq_search、gain_search、fit）整理欄位，但 builder/runtime/remote 與 persisted workflow/run workflow snapshot 仍讀寫 flat logical keys。visible label 是 presentation：group 已承載的 domain prefix 會省略，且初始值欄位顯示為 `initial_*` / `target_*` wording；field key / logical key / persisted raw 不縮短。`ScalarSpec.tooltip` / `SweepSpec.tooltip` / `CenteredSweepSpec.tooltip` 是 spec-level hover description source，`CfgFormWidget` 會把它併入 default `FieldDecoration.tooltip`，app-local decoration provider 仍可覆寫。strategy/mode group 用 shared `ChoiceSectionSpec` 描述 selector 與各 choice 需要的欄位，value tree 仍保留完整 union；feedback slot 的 disabled state 併入 `strategy=off`，estimator `idw` 顯示 `idw_k` / `idw_epsilon` / `decay_points`，`last_good` 只顯示 `decay_points`，controller `log_step` 只顯示 strategy，step gain/decay points 使用 slot default constants。其它 mode 如 fixed range / fixed relax 會隱藏其自動生成參數。Builder 用 `override_plan(schema)` 宣告 Generation 會覆寫的 Default cfg path；實驗 node 用 `experiments/_support/utils/override_plan.py` 的 `NodeOverridePlan` 線性宣告 module dependency 與 generated path，底層 `OverridePlan` contract 不變。`OverridePlan` 只描述 Default cfg path，不描述 `generation.*` 欄位，run start materialize `RunCfgSnapshot(base_cfg, override_plan, knobs)` 並驗證 path 存在於 `base_cfg` 且不是 whole-module replacement。每個 flux point 從 run-start `base_cfg` deep-copy 後只套用 builder-declared patches；undeclared path、first point 的 `after_first_point` patch、缺失 non-fallback path 都 Fast Fail。Runtime overlay path 只覆蓋 dependency source 實際接管的 core leaves；所有 autofluxdep readout refs restrict to `Pulse Readout`，使 readout freq/gain/window path 在 UI、OverridePlan 與 runtime patch 中保持同一 shape。readout dependency 以 editable template overlay 呈現：freq/gain/length leaves 是 fallback policy，有 runtime dependency 值時覆寫，沒有時保留 Default cfg；channel、trigger offset、waveform style 等其它 module 欄位仍讀 Default cfg，parent module row 顯示 template badge。shared `CfgFormWidget`依 full path 套用 `FieldDecoration`；node cfg form 會把 `all_points` 欄位標成 generated、disabled，把 `after_first_point` 欄位標成 initial，把 `fallback` 欄位標成 fallback 並保持可編輯。lenrabi/T1/T2/T2Echo 的 auto time sweep 只生成 `sweep.length.stop`；`sweep.length.start` 與 `expts` 保持 Default cfg 可編輯，且 start 是下層 `start_us` 來源；lenrabi 不再有獨立 `stop_min_us`，避免與 Default cfg 的 sweep start 形成雙重下限。T1/T2/echo 預設用 `t1_seed_us` / `t2*_seed_us` 加 factor 產生 relax delay 與 sweep stop，其中 T1/T2/T2Echo 的 auto sweep stop 再由 `max_length` 套上上限（fresh default 對齊 seed × factor）。T2Echo 的 detune ratio 對齊 adapter 預設 `0.1`，fit method 預設 `auto_by_detune`。T1/T2/T2Echo scalar feedback 只在 residual gate 與 sweep-window bound 可信時產出，否則保留 raw Result row 但不送出 `t1` / `t2r` / `t2r_detune` / `t2e` Patch key。smoothed dependency projection 會跨 raw-absent 點 carry forward 上一個可信 smoothed 值，但 smoother state 本身不因缺 raw 而 advance。lenrabi 是以 `expected_pi_length` 為 setpoint 的 drive-gain 負回饋控制：第一點或 `strategy=off` slot 回到 seed/smoothed `pi_product` 行為；可信 fit 後 generic `log_step` controller 依 `log(measured_pi_length / expected_pi_length)` 對下一點 gain 提 proposal，預設 step gain 0.5 與 decay 3.0，下一點以 confidence 在 log-domain 平滑混合 proposal 與 open-loop `pi_product` gain；`pi_product_factor` 固定為 1.2，drive gain cap 固定為 `1.0` 並由 lenrabi use site 處理，不作為 Generation 欄位；每個可信點仍產出 raw `pi_product = measured_pi_length * applied_gain` 供既有 smoothing/diagnostic 使用。ro_optimize 用 T1 seed 設 relax delay，readout freq/gain window 可在 fixed sweep 與 previous-best centered window 間切換；t2echo 可依 detune 自動選 fringe/decay fit 或手動固定 fit method。
- **T1 sweep distribution**：T1 node 的 Generation `uniform` 預設為 `True`，沿用線性
  hardware sweep；設為 `False` 時仍使用 Default cfg 的 `sweep.length.start` / `stop` /
  `expts` 與既有 auto/fixed stop、`max_length` 規則，但 runtime 以 non-uniform delay table
  取得更密集的 early-decay 取樣。
- **ro_optimize landscape**：readout freq × gain SNR map 在 argmax 前使用共用 wavelet smoothing；freq/gain 軸由 `make_cfg()` 的 search window 決定，`produce()` 會在 acquire 前同步 result/plot 軸與實際 sweep；fresh default 以 md `r_f` 或 readout module freq/gain 為中心，gain fallback 為 `center=0.1, span=0.2`（lower 後 `0.0..0.2`），使用窄窗做 first-pass search。previous-best search window 的 Default cfg `sweep.freq` / `sweep.gain` 以 centered value model 顯示，是 first-point initial field，UI 保持可編輯；第二點起才由 previous best 生成 centered window。Patch emission 前會拒絕 invalid/flat/non-unique/boundary/low-prominence landscape；被拒絕的 row 保留 SNR map evidence，但不送出 `best_ro_freq` / `best_ro_gain` / `opt_readout`，讓 downstream 使用最近可信值或既有 fallback。plotter 以 scatter marker 標註目前 row 以前最近一次 finite `best_ro_freq` / `best_ro_gain`。此 node 仍是 no-fit argmax，可信時產出 `best_ro_freq` / `best_ro_gain` / `opt_readout`。
- **produce 例外保護**：`orchestrator.run` 只透過 `RunObserver` 回報 node/skip/row/commit/failure 事件；catch `produce` 例外 → 記 `self.run_error`＋停 sweep；controller 檢查後 emit `RunFailedPayload(message, node, flux_idx, stage)`，finalize/pause/stop artifact failure 也用同一 payload 帶 `stage`。UI `MainWindow._on_run_failed` 解鎖＋彈 warning；running-close terminal stop path 不重複彈 warning。真 GUI run 的 node 例外不會 crash app。
- **predictor 兩層 + live view**：`exp_context.predictor` 存 raw `FluxoniumPredictor`（PredictorDialog/`PredictorService` 載——predictor seam 自 ConnectionService 拆進 PredictorService，ADR-0026 §5）；setup project 若 resolved `params.json` 含 `fluxdep_fit`，controller best-effort 安裝 predictor，使用者仍可在 PredictorDialog typed path/Browse 載入或手動覆蓋；workflow memento 保存 raw active predictor model 的 numeric fields（EJ/EC/EL、flux anchors、bias；path 只作 provenance/prefill），restore 以 `SetModelParamsRequest` 重建，不把 path 當 canonical reload source。`_build_tools` 每 run wrap 成 base predictor `FluxoniumPredictorAdapter`（None→`SimplePredictor`），stash 進 run-lived `state.run_predictor`；qubit_freq 的 physical overlay 與 generic residual correction 是 run-local state，分別放在 `Tools` recovery state 與 `Tools.feedback` slot。run 中 Predictor toolbar button 保持可開，shared `PredictorDialog` 以 persistent close hide/reuse，並保存 typed params path、tracked transitions、selected tab 等 dialog UI preference；進 `set_live_mode(True)` 時模型欄位、Load/Browse/Apply、device-value picker 與 marker drag 變 read-only，marker 由目前 flux point 的 device value 驅動。qubit_freq row committed 後 controller 只 emit shared `PredictorChangedPayload` 讓 dialog 重讀 raw predictor / redraw curves；dialog 不讀 run-local physical overlay 或 residual correction。Run finish/stop/fail 會退出 live mode。**勿混** measure 的 freq-預測 predictor。
- **node 參數 SSOT**：`PlacedNode` 持有 per-placement `NodeCfgSchema` value tree；fresh placement 以 active `ExpContext` 建預設值，persisted restore 以保存的 raw value tree 為準。`NodeSchemaBuilder` 宣告 cfg shape、logical projection 與 generation choices，`NodeCfgSchema.logical_paths` 將 UI tree 投影成 runtime/remote/persistence 的 stable logical keys。UI form 經 `controller.set_node_cfg_value()` 一次提交完整 typed value tree，主執行緒再 clear run products、bump workflow version、emit event 與 persist；run start materialize `base_cfg` / `override_plan` / `knobs`，node 以 `RunEnv.knob()` / `knobs_view()` 讀 canonical snapshot，並由 `Builder.point_cfg()` 建立各 flux point 的可變 cfg。
- **cfg commit 與 run snapshot**：Controller 只接受 `set_node_cfg_value()` 的完整 typed value tree；tests 以 test-local draft 寫入，沒有 logical-key façade。production session 與未注入 `ml` 的 dry run 都在 lowering 前 clone 完整 `ModuleLibrary`，同一 clone 供 `base_cfg` / `knobs` snapshot、`RunEnv.make_cfg`、waveform lookup 與 module fallback 使用。`RunCfgSnapshot` 以 closed-world representation 保存 run-start truth：遞迴凍結 containers、以 immutable carrier 保存 `SweepCfg`、以 bytes-backed read-only view 保存 ndarray，unknown leaf Fast Fail。`RunEnv.knob()` / `knobs_view()` 沿用同一 frozen mapping，不在熱讀路徑複製；只有 `knobs()`、point cfg 或 wire materialization 才還原真正 `SweepCfg` 與 writable ndarray，因此 caller、不同 flux point 與 artifact output 都不能污染 canonical snapshot。
- **workflow persistence + close behavior**：對齊 measure-gui 的 Memento + Caretaker：`Controller` 是 originator，`services/caretaker.py` 只負責單檔 JSON I/O（`autofluxdep_state_v1.json`，user cache 的 gui 目錄），workflow/node/flux/UI preference edits 會 debounce autosave。`MainWindow.closeEvent` 在一般 live operations 上仍走 shared shutdown driver；running RUN 則先確認、送 terminal stop、等待最多 30 秒，逾時只顯示 explicit Force Close；paused close 需確認 discard/terminal finalize，並把 artifact finalize 為 stopped 後丟棄 in-memory paused session。operation-running close confirmations 使用 shared `DialogPresenter` 的 callback-style dialogs，不用 blocking `exec()`/modal return value。這些 app-shell information/warning/confirm/destructive-confirm 行為經 shared `DialogPresenter` 注入，production 用 Qt dialog，headless tests 用 recording presenter script decision。Force Close 可能只留下最後已 flushed 的 artifact state。RUN finished/stopped/failed terminal callback 會立即 flush，避免長掃描或 stop 後遺失 workflow edits；capture/write 失敗會轉成 typed persistence failure 並由 controller 記錄，不讓 Qt timer slot abort GUI。memento 保存 shared startup prefill（scope id / chip-qubit-res / connection / remembered devices）、Nodes（type/name/enabled + `NodeCfgSchema.to_persisted_raw()`；Default cfg 走 `zcu_tools.gui.cfg` codec raw shape，`generation` 以 flat logical-key raw map 保存）、flux sweep（三個 expression 字串 + 最後 commit 的 explicit values）、raw active predictor model（numeric fields；path only provenance/prefill）與 app-shell UI preference（auto-follow active tab、Predictor dialog typed path/tracked transitions/tab）。不保存 run result、plot、`state.run_predictor`、run-local feedback/correction estimator、SoC、live mode、device live value、selected device、cached hardware value 或 canvas arrays/status。restore 只 seed setup dialog prefill 與 remembered devices，不自動套用 project 或連線；單一壞 node 或 predictor slice restore 會被 reject 並保留其它 workflow/flux state。
- **ModuleLibrary fallback aliases**：workflow 內部的 module keys 維持 domain logical name（`readout`、`opt_readout`、`pi_pulse`、`pi2_pulse`），但 `ModuleDep.aliases` 讓 library fallback 使用 measure-gui calibrated names：readout 類找 `readout_dpm → readout_rf → readout → res_readout`，pi 類找 `pi_len → pi_amp → pi_pulse`，pi2 類找 `pi2_amp → pi2_len → pi_amp → pi_len → pi2_pulse`。Orchestrator 在建立時只快照 providers 實際宣告的 library names/aliases；snapshot 與每次 consumer resolution 都 deep copy，缺 key 保持 missing 語意。Node-produced module 仍永遠優先於 ML fallback，`RunEnv.ml` 則只負責完整 library 的 cfg lowering。fresh lenrabi Default cfg 的 editable `Rabi Pulse` seed 同樣優先使用 ML `pi_len → pi_amp → pi_pulse`，讓 length-Rabi 預設從已校準的 length pulse 出發；若 library key 不存在才保留 adapter blank pulse。
- **Produced module contract**：workflow module stream 只流通可被 `ModuleCfgFactory` lower 的 concrete module；lenrabi 產出的 `pi_pulse` / `pi2_pulse` 是從 lowered drive pulse 複製並覆寫 waveform length 的 `type: "pulse"` module，不使用 `type: "pi"` / `"pi2"` 這類 domain placeholder。需要 concrete pulse 的 downstream node 以 required `ModuleDep` 表達需求；resolver 可依 declaration 使用 latest-produced、current-point-only、library fallback 或 no-fallback。T1 宣告 current-point-only `pi_pulse` 且不使用 ModuleLibrary fallback；lenrabi 宣告 current-point-only `qubit_freq`，所以本點 `qubit_freq` fit 不可信時本點 lenrabi/T1 會 skip，而不是混用上一點或 library pulse。lenrabi fit 先比較 decay/non-decay Rabi fit 並只在 pi length、boundary、residual gate 可信時送出 feedback Patch；不可信點保留 raw Result row 但不產生 `pi_length` / `pi_product` / drive modules。pi2 過短或 non-finite 時仍送出 `pi_length` / `pi_product` scalar feedback，但不產生 `pi2_length` 或任何 drive module，避免下游混用不同 flux point 的 pulse pair。
- **flux sweep input**：左側 flux start/stop/npts 復用 shared `ScalarField` + `ScalarWidget`；`AutofluxCfgBindings` 提供app-local expression、option與reference policy，field支援direct value、explicit expression mode與ghost preview。Run click 由 controller 以 active md resolve 並 commit 成 explicit `np.linspace` values，三個 expression 字串會進 workflow memento。Run button 以 controller `run_readiness()` 判斷 setup/project/workflow/enabled-node readiness；flux source 可不選，`flux_device_name is None` 表示 bare-number sweep。左側 node checkbox 只控制 future run membership，disabled node 仍可選取與編輯 cfg。node list row 的 checkbox + label 由自訂 row widget 單一繪製，`QListWidgetItem` 只保存 tooltip / data，不放 display text，避免 Qt delegate 與 row label 疊字。
- **auto-follow tab policy**：run 中進入新的 workflow node 時，左側 selection 會跟著切到該 node 並顯示 run tab；如果進入的是目前已選取的 node，保留使用者當下的 edit/run sub-tab，不打斷正在查看的設定頁。左側 checkbox 可在 run 中即時開關此自動 selection/tab switch；重新打開時會立刻跟到目前 active node，且 preference 會保存。auto-follow navigation 只更新 selection/title/run canvas；edit form 在使用者打開 edit tab 時才 lazy materialize，避免 run 中每次 active-node transition 都重建 hidden typed form。run 期間已 materialize 的 node edit form 會以 node/index 快取並重用；run 結束時不切換使用者目前的 edit/run sub-tab，只清掉 hidden form cache 並回到一般 workflow 編輯語意。run 中若使用者手動切換 node 或 edit/run tab，auto-follow 會自動取消並同步取消 checkbox，避免下一個 node 事件把視圖拉走。
- **run progress render**：run 進度由共用 `ProgressStack` 顯示 per-node rounds，底部 global bar 顯示 flux sweep；兩者都保留 profiling probe。node-level Schedule acquire 使用 close-on-finish 的 rounds bar，避免每個 flux row 的 bar 殘留累積。高頻 progress event 只在 maximum/format/value 實際變更時更新 lightweight progress widget，`setValue` 只寫狀態並排程 repaint，避免主線程卡在 native `QProgressBar.setValue`。

- **read-only remote bridge**：`services/remote/` 是 autofluxdep 的第二 View（RPC face onto Controller，ADR-0013），對標 fluxdep/dispersive bridge。完全 read-only——agent 只觀測、user 在 GUI 驅動 workflow，**無 stop、continue、restart、close、零 mutation**。`RemoteReadModel` 是 handler 的唯一 app read facade；`dispatch.py` 只負責 method routing 與 screenshot 檔案輸出，不直接理解 Controller/state/node/result 細節。三 lifecycle（launch/connect/disconnect）+ 6 agent-facing 純查詢：`state.check`（readiness flags + `is_running` / `is_paused` / `next_flux_idx` / `run_status` + predictor 旗標）/`project.info`（chip/qub/result_dir/database_path/params_path，None→五欄 null）/`workflow.list`（每 placed node 的 name/type/enabled/provides/requires/has_result，**不含 predictor service**——run 時才 prepend；disabled node 仍列出但 future run 不執行）/`node.cfg(name)`（單 node 未 lower 的 user knobs，仍是 flat logical-key projection；`SweepSpec`→`{start,stop,expts}`，`CenteredSweepSpec`→`{center,span,expts,step}`，走 `NodeCfgSchema.read_knobs()`；另回傳 builder `override_plan`）/`result.summary`（每有 Result 的 node 的 n_flux/n_measured/tiny fit_summary，**非 raw 2D**；`n_measured` 數 primary raw signal finite rows，`fit_summary.n_fitted` 數 fit scalar finite rows，與 run artifact 的 `result_progress_summary()` 共用 declaration）/`ui.screenshot(target="window")`（只抓主視窗 PNG 到 `/tmp` 並回傳 path/bytes，純觀測）。`resources.versions` 只是 MCP/RPC optimistic-concurrency bookkeeping，存在於 registry/METHOD_SPECS，不是 agent-facing pure query tool，也走 `RemoteReadModel`。`AutoFluxDepGuiBehavior` 在 `gui.runtime` 內建立 app-local adapter；runtime 負責 Agg-only plotting policy、control option construction、adapter start/stop 與 bind failure exit code。所有 handler 走 base 主執行緒 marshal；run 中讀 `run_results` 拿一致快照（worker 原地填 numpy 非 State semantic write）。MCP server 在 `lib/zcu_tools/mcp/autofluxdep/`（port 8768），skill `run-autofluxdep-gui`。
