# Arbitrary waveform assets allow optional formula recipes

Arbitrary waveform data is modeled as a qubit-scoped single-file `.npz` asset containing required playback arrays and an optional formula recipe. The required arrays are `idata`, `qdata`, and `time`; a formula recipe, when present, is stored as a single UTF-8 JSON scalar field named `recipe_json`. No derived metadata fields are persisted: duration, sample count, Q presence, peak magnitude (`peak_abs`), and preview summaries are computed on demand from the arrays or recipe; normalization scale is a transient render value and is not exposed as persisted metadata or GUI detail. Playback payload validation is strict for both formula-rendered and imported assets: the archive may contain only `idata`, `qdata`, `time`, and optional `recipe_json`; unknown keys fail with reason `unknown_npz_key`; if `recipe_json` is present but invalid, the asset fails fast instead of being treated as a raw asset. `idata`, `qdata`, and `time` must be same-shape 1D numeric arrays with length at least 2 and at most `MAX_ARB_WAVEFORM_SAMPLES = 1_000_001`, all values must be finite, `time[0]` must be `0`, `time` must be strictly increasing in microseconds, and waveform magnitude `sqrt(idata**2 + qdata**2)` must stay in normalized range `[0, 1]`. Imported raw `.npz` time axes do not need uniform spacing; strict monotonicity is enough. Assets over the sample-count limit fail fast with reason `sample_count_too_large` before save/import/preview work proceeds. Each asset is addressed by an Arbitrary Waveform Data Key matching `^[A-Za-z][A-Za-z0-9_]*$`, and the file path is `<data_key>.npz`. User and agent-created formula waveforms embed the recipe so they can be loaded and modified, but imported raw waveform data may remain valid without one. External file import is limited to the `.npz` NumPy archive format with required `idata`/`qdata`/`time` fields; `.npy`, CSV, clipboard, text tables, and arbitrary column mapping are not supported formats. The repository only reads and writes the new `<data_key>.npz` layout; the older `<data_key>/data.npz` shape has no runtime compatibility path because no existing production data needs migration. This keeps raw experimental waveform imports possible while still giving formula-authored assets an agent-friendly representation without splitting one waveform asset across multiple files.
